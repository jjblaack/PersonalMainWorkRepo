## 1. 权限系统概述

### 1.1 权限系统的职责

权限系统负责控制 AI 工具的执行权限，确保：

- 敏感操作需要用户审批
- 不同风险级别的操作有不同的权限要求
- 用户可以自定义权限策略
- 防止意外或恶意的操作

### 1.2 权限系统架构


## 2. 权限检查机制

### 2.1 权限检查流程

```typescript
// src/hooks/toolPermission/ (推断)

async function checkToolPermission(
  tool: Tool,
  input: unknown,
  context: ToolContext
): Promise<PermissionResult> {
  // 1. 获取工具的权限规格
  const permissionSpec = tool.getPermission?.(input, context) 
    || getDefaultPermission(tool);
  
  // 2. 获取当前权限模式
  const mode = context.permissionMode;
  
  // 3. 检查是否 bypass 模式
  if (mode === 'bypassPermissions') {
    return { granted: true };
  }
  
  // 4. 检查是否 ACCEPT_ALL 模式
  if (mode === 'ACCEPT_ALL_TOOLS') {
    return { granted: true };
  }
  
  // 5. 检查 auto 模式
  if (mode === 'auto') {
    // 低风险工具自动执行
    if (permissionSpec.level === 'low') {
      return { granted: true };
    }
    // 高风险工具仍需要审批
    if (permissionSpec.level === 'high') {
      return {
        granted: false,
        requiresUserApproval: true,
      };
    }
  }
  
  // 6. 检查用户审批历史
  const approvalHistory = await getToolApprovalHistory(tool.name);
  if (approvalHistory.alwaysAllow) {
    return { granted: true };
  }
  
  // 7. 默认需要审批
  return {
    granted: false,
    reason: 'Requires user approval',
    requiresUserApproval: true,
  };
}
```

## 3. 权限模式

### 3.1 权限模式类型

```typescript
// src/types/permissions.js (推断)

export type PermissionMode =
  | 'default'              // 默认模式：所有操作需要审批
  | 'plan'                 // 计划模式：批量审批
  | 'auto'                 // 自动模式：低风险工具自动执行
  | 'bypassPermissions'    // 开发模式：绕过所有审批
  | 'ACCEPT_ALL_TOOLS'     // 接受所有工具
  | 'testing'              // 测试模式
```

### 3.2 模式详解

#### 3.2.1 default (默认模式)

**行为**: 所有工具调用都需要用户审批

**使用场景**: 
- 初次使用
- 高安全性要求环境

#### 3.2.2 plan (计划模式)

**行为**: 用户可以预先审批一系列操作

**使用场景**:
- 已知的多步骤任务
- 批处理操作

#### 3.2.3 auto (自动模式)

**行为**: 
- 低风险工具 (读取、搜索) 自动执行
- 中风险工具 (编辑) 需要审批
- 高风险工具 (删除、系统命令) 需要审批

**使用场景**:
- 日常开发
- 信任的会话

#### 3.2.4 bypassPermissions (绕过权限)

**行为**: 所有工具自动执行，无需审批

**使用场景**:
- 开发和测试
- 完全信任的环境

**警告**: 仅在安全环境中使用

### 3.3 模式切换

```bash
# CLI 选项
claude --permission-mode auto

# 命令切换
/permission-mode auto

# 运行时切换 (对话框)
```

```typescript
// 审批决策 (推断)

async function handlePermissionDecision(
  toolName: string,
  decision: 'approve' | 'deny' | 'always_allow'
): Promise<void> {
  switch (decision) {
    case 'approve':
      // 仅此一次批准
      await recordApproval(toolName, 'once');
      break;
      
    case 'always_allow':
      // 总是允许
      await recordApproval(toolName, 'always');
      break;
      
    case 'deny':
      // 拒绝，记录到拒绝历史
      await recordDenial(toolName);
      break;
  }
}
```

### 4.3 审批历史管理

```typescript
// 审批历史存储 (推断)

interface ApprovalHistory {
  toolName: string;
  decisions: Array<{
    timestamp: number;
    decision: 'approve' | 'deny';
    input: unknown;
  }>;
  alwaysAllow?: boolean;
  alwaysDeny?: boolean;
}

async function getToolApprovalHistory(toolName: string): Promise<ApprovalHistory> {
  const history = await loadApprovalHistory();
  return history[toolName] || { toolName, decisions: [] };
}

async function recordApproval(toolName: string, type: 'once' | 'always'): Promise<void> {
  const history = await loadApprovalHistory();
  
  if (!history[toolName]) {
    history[toolName] = { toolName, decisions: [] };
  }
  
  if (type === 'always') {
    history[toolName].alwaysAllow = true;
  }
  
  history[toolName].decisions.push({
    timestamp: Date.now(),
    decision: 'approve',
    input: null,
  });
  
  await saveApprovalHistory(history);
}
```

## 5. 权限 Hook

### 5.1 useCanUseTool Hook

**位置**: `src/hooks/useCanUseTool.js`

**功能**: 检查工具是否可用

```typescript
// src/hooks/useCanUseTool.js (推断)

export function useCanUseTool(): CanUseToolFn {
  const { permissionMode } = usePermissionContext();
  const { approvalHistory } = useApprovalHistory();
  
  const canUseTool = useCallback(async (tool: Tool, input: unknown): Promise<boolean> => {
    // 1. 检查 bypass 模式
    if (permissionMode === 'bypassPermissions') {
      return true;
    }
    
    // 2. 检查总是允许
    if (approvalHistory[tool.name]?.alwaysAllow) {
      return true;
    }
    
    // 3. 检查总是拒绝
    if (approvalHistory[tool.name]?.alwaysDeny) {
      return false;
    }
    
    // 4. 检查工具权限级别
    const permissionSpec = tool.getPermission?.(input) || getDefaultPermission(tool);
    
    if (permissionMode === 'auto' && permissionSpec.level === 'low') {
      return true;
    }
    
    // 5. 需要用户审批
    return false;
  }, [permissionMode, approvalHistory]);
  
  return canUseTool;
}
```

### 6.2 工具权限定义

```typescript
// 工具权限定义示例 (推断)

class BashTool extends Tool {
  name = 'Bash';
  
  getPermission(input: { command: string }): PermissionSpec {
    const command = input.command.toLowerCase();
    
    // 检测危险命令
    const dangerousCommands = ['rm -rf', 'mkfs', 'dd', 'format'];
    const isDangerous = dangerousCommands.some(cmd => command.includes(cmd));
    
    return {
      level: isDangerous ? 'high' : 'medium',
      requiresApproval: true,
      blockedCommands: dangerousCommands,
    };
  }
}

class FileReadTool extends Tool {
  name = 'FileRead';
  
  getPermission(input: { file_path: string }): PermissionSpec {
    const path = input.file_path;
    
    // 检查敏感路径
    const sensitivePaths = ['/etc', '/proc', '/sys', 'C:\\Windows\\System32'];
    const isSensitive = sensitivePaths.some(p => path.startsWith(p));
    
    return {
      level: isSensitive ? 'high' : 'low',
      requiresApproval: isSensitive,
      blockedPaths: sensitivePaths,
    };
  }
}
```

## 7. 安全边界

### 7.1 文件系统边界

```typescript
// 文件系统安全检查 (推断)

function isPathAllowed(filePath: string, allowedPaths: string[]): boolean {
  const resolvedPath = path.resolve(filePath);
  
  // 检查是否在允许的路径内
  for (const allowedPath of allowedPaths) {
    if (resolvedPath.startsWith(path.resolve(allowedPath))) {
      return true;
    }
  }
  
  return false;
}

function isPathBlocked(filePath: string, blockedPaths: string[]): boolean {
  const resolvedPath = path.resolve(filePath);
  
  for (const blockedPath of blockedPaths) {
    if (resolvedPath.startsWith(path.resolve(blockedPath))) {
      return true;
    }
  }
  
  return false;
}
```

### 7.2 命令执行边界

```typescript
// 命令安全检查 (推断)

const BLOCKED_PATTERNS = [
  /^rm\s+(-[rf]+\s+)?\//,     // rm -rf /
  /^mkfs/,                    // 格式化
  /^dd\s+of=/,                // dd 写入
  /^chmod\s+(-R\s+)?777/,     // chmod 777
  /^: fork\(\)\s*\{/,         // fork bomb
  /curl.*\|\s*bash/,          // curl | bash
  /wget.*\|\s*bash/,          // wget | bash
];

function isCommandSafe(command: string): boolean {
  for (const pattern of BLOCKED_PATTERNS) {
    if (pattern.test(command)) {
      return false;
    }
  }
  
  return true;
}
```

### 7.3 网络访问边界

```typescript
// 网络访问检查 (推断)

const BLOCKED_HOSTS = [
  'localhost',
  '127.0.0.1',
  '169.254.169.254',  // AWS metadata
  'metadata.google.internal',  // GCP metadata
];

function isUrlAllowed(url: string, allowedHosts: string[]): boolean {
  const parsed = new URL(url);
  
  // 检查阻止的主机
  if (BLOCKED_HOSTS.some(host => parsed.hostname === host || parsed.hostname.endsWith(`.${host}`))) {
    return false;
  }
  
  // 检查允许的主机列表
  if (allowedHosts.length > 0) {
    return allowedHosts.some(host => parsed.hostname === host || parsed.hostname.endsWith(`.${host}`));
  }
  
  return true;
}
```

```typescript
interface DenialRecord {
  toolName: string;
  timestamp: number;
  input: unknown;
  reason?: string;
}

async function recordDenial(toolName: string, input: unknown, reason?: string): Promise<void> {
  const denials = await loadDenials();
  
  denials.push({
    toolName,
    timestamp: Date.now(),
    input,
    reason,
  });
  
  // 只保留最近的 100 条
  if (denials.length > 100) {
    denials.shift();
  }
  
  await saveDenials(denials);
}

async function getRecentDenials(toolName: string, limit = 10): Promise<DenialRecord[]> {
  const denials = await loadDenials();
  return denials
    .filter(d => d.toolName === toolName)
    .slice(-limit);
}
```

## 总结

Claude Code 的权限系统提供了多层次的安全保护：

1. **权限级别**: 低/中/高风险分类
2. **权限模式**: 多种模式适应不同场景
3. **用户审批**: 灵活的审批机制
4. **安全边界**: 文件系统、命令、网络边界
5. **拒绝跟踪**: 记录和分析拒绝历史

这个系统确保了 AI 工具在提供强大功能的同时，不会对系统造成意外损害。
