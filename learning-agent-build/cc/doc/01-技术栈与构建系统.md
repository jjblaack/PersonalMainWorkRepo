## 1. Runtime 环境 - Bun

### 1.1 为什么选择 Bun？

Bun 是一个现代化的 JavaScript 运行时，旨在替代 Node.js，提供显著的性能提升和更好的开发者体验。

**Bun 的核心优势**：

| 特性 | 说明 | 对 Claude Code 的价值 |
|------|------|------|
| **启动速度** | 比 Node.js 快 4 倍 | CLI 工具需要快速响应 |
| **原生 TypeScript** | 无需转译，直接运行 TS | 简化构建流程 |
| **内置打包器** | `bun build` 提供快速打包 | 单一可执行文件分发 |
| **API 兼容性** | 兼容 Node.js API | 现有生态无缝迁移 |
| **内存效率** | 更低的内存占用 | 长时间运行会话更稳定 |

### 1.2 Bun 特有 API 使用

#### 1.2.1 `bun:bundle` 特性标志

```typescript
// src/main.tsx
import { feature } from 'bun:bundle';

// 构建时死代码消除
const coordinatorModule = feature('COORDINATOR_MODE') 
  ? require('./coordinator/coordinatorMode.js')
  : null;

const assistantModule = feature('KAIROS') 
  ? require('./assistant/index.js')
  : null;
```

**工作原理**：
- `feature('FLAG_NAME')` 在构建时被解析为布尔值
- 未启用的功能代码被完全消除，不打包到最终产物
- 比运行时条件判断更高效

#### 1.2.2 主要特性标志

```typescript
// 已知特性标志列表
- 'PROACTIVE'         // 主动模式
- 'KAIROS'            // 助手模式 (Assistant)

- 'VOICE_MODE'        // 语音输入模式
- 'AGENT_TRIGGERS'    // 代理触发器
- 'MONITOR_TOOL'      // 工具监控
- 'COORDINATOR_MODE'  // 协调器模式
```

## 6. 代码搜索 - ripgrep

### 6.1 集成方式

ripgrep (rg) 是一个基于 Rust 的快速文本搜索工具，Claude Code 将其集成用于 GrepTool。

```typescript
// src/tools/GrepTool/GrepTool.ts (推断)

import { spawn } from 'child_process';

export async function runRipgrep(
  pattern: string,
  options: {
    path?: string;
    glob?: string;
    caseSensitive?: boolean;
    maxResults?: number;
  }
): Promise<GrepResult> {
  const args = ['--json', '--no-heading'];
  
  if (options.glob) {
    args.push('--glob', options.glob);
  }
  
  if (!options.caseSensitive) {
    args.push('-i');
  }
  
  if (options.maxResults) {
    args.push('--max-count', options.maxResults.toString());
  }
  
  args.push(pattern);
  if (options.path) args.push(options.path);
  
  return new Promise((resolve, reject) => {
    const proc = spawn('rg', args);
    const results: GrepMatch[] = [];
    
    proc.stdout.on('data', (data) => {
      for (const line of data.toString().split('\n')) {
        if (line) {
          results.push(JSON.parse(line));
        }
      }
    });
    
    proc.on('close', () => resolve({ matches: results }));
    proc.on('error', reject);
  });
}
```

### 6.2 性能优化

- **二进制工具**: ripgrep 是编译型语言，比 grep 快 10-100 倍
- **并行搜索**: 自动利用多核
- **智能忽略**: 自动跳过 .git, node_modules 等
- **流式处理**: 无需等待全部结果

## 9. 认证系统

### 9.1 OAuth 2.0 流程

```typescript
// src/services/oauth/flow.ts (推断)

class OAuthFlow {
  async startAuthFlow() {
    const config = await getOauthConfig();
    const state = generateRandomState();
    const codeVerifier = generateCodeVerifier();
    
    // 存储 PKCE 参数
    await secureStorage.set('oauth_state', state);
    await secureStorage.set('code_verifier', codeVerifier);
    
    const authUrl = new URL(config.authorizationEndpoint);
    authUrl.searchParams.set('client_id', config.clientId);
    authUrl.searchParams.set('redirect_uri', 'http://localhost:8989/callback');
    authUrl.searchParams.set('response_type', 'code');
    authUrl.searchParams.set('scope', 'openid profile email');
    authUrl.searchParams.set('state', state);
    authUrl.searchParams.set('code_challenge', await generateCodeChallenge(codeVerifier));
    authUrl.searchParams.set('code_challenge_method', 'S256');
    
    // 在浏览器中打开授权页面
    await open(authUrl.toString());
    
    // 启动本地服务器接收回调
    return this.waitForCallback();
  }
  
  async exchangeCodeForToken(code: string, receivedState: string) {
    const storedState = await secureStorage.get('oauth_state');
    if (receivedState !== storedState) {
      throw new Error('State mismatch');
    }
    
    const codeVerifier = await secureStorage.get('code_verifier');
    
    const response = await fetch(config.tokenEndpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        grant_type: 'authorization_code',
        code,
        redirect_uri: 'http://localhost:8989/callback',
        code_verifier: codeVerifier,
      }),
    });
    
    return response.json();
  }
}
```

### 9.2 JWT 使用

```typescript
// src/bridge/jwtUtils.ts (推断)

import { SignJWT, jwtVerify } from 'jose';

const encoder = new TextEncoder();

export async function createSessionToken(payload: {
  sessionId: string;
  userId: string;
  expiresAt: number;
}): Promise<string> {
  const secret = encoder.get(await getSessionSecret());
  
  return new SignJWT(payload)
    .setProtectedHeader({ alg: 'HS256' })
    .setIssuedAt()
    .setExpirationTime(payload.expiresAt)
    .sign(secret);
}

export async function verifySessionToken(token: string) {
  const secret = encoder.get(await getSessionSecret());
  
  try {
    const { payload } = await jwtVerify(token, secret);
    return payload;
  } catch {
    return null;
  }
}
```

```typescript
export async function getKeychainItem(service: string, account: string): Promise<string | null> {
  try {
    const result = execSync(`security find-generic-password -s "${service}" -a "${account}" -w`, {
      encoding: 'utf-8',
      stdio: ['pipe', 'pipe', 'ignore'],

  } catch {
    return null;
  }
```

### 10.4 Source Map 问题

**暴露原因**：
```javascript
// 构建配置可能的问题
--sourcemap=external  // 生成独立的 .map 文件
```

**风险**：
- .map 文件包含完整源代码
- 未混淆的 TypeScript 源码
- 内部实现细节暴露

**建议**：
```bash
# 生产环境不应发布 source map
bun build --sourcemap=none
# 或限制访问
bun build --sourcemap=external
# 并在发布时排除 .map 文件
```
