## 1. 状态管理架构概述

### 1.1 分层状态设计

Claude Code 采用三层状态管理架构：

```
┌─────────────────────────────────────────────────────────────┐
│  全局状态层 (bootstrap/state.ts)                             │
│  - 会话级单例状态                                            │
│  - 性能指标和遥测                                            │
│  - 会话元数据                                                │
│  - 访问器模式 (getter/setter)                                │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  应用状态层 (state/AppStateStore.ts)                         │
│  - React 组件全局状态                                         │
│  - 设置和配置                                                │
│  - MCP/插件状态                                              │
│  - 任务和通知                                                │
│  - Store 模式 (Zustand 风格)                                  │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  上下文层 (context/)                                         │
│  - 领域特定上下文                                            │
│  - MailboxContext (组件间通信)                               │
│  - ModalContext (模态框管理)                                 │
│  - NotificationsContext (通知系统)                           │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 状态类型对比

| 层面 | 访问模式 | 更新机制 | 典型用途 |
|------|----------|----------|----------|
| 全局状态 | Getter/Setter | 直接修改 | 会话 ID、成本追踪、遥测 |
| 应用状态 | Selector | setState(updater) | 设置、任务、消息 |
| 上下文 | useContext | Context Provider | 模态框、通知、领域数据 |

### 3.1 Store 实现

**文件**: `src/state/store.ts`

```typescript
// Zustand 风格的简单 Store 实现

type Listener = () => void
type OnChange<T> = (args: { newState: T; oldState: T }) => void

export type Store<T> = {
  getState: () => T
  setState: (updater: (prev: T) => T) => void
  subscribe: (listener: Listener) => () => void
}

export function createStore<T>(
  initialState: T,
  onChange?: OnChange<T>,
): Store<T> {
  let state = initialState
  const listeners = new Set<Listener>()

  return {
    getState: () => state,

    setState: (updater: (prev: T) => T) => {
      const prev = state
      const next = updater(prev)
      if (Object.is(next, prev)) return  // 跳过无变化更新
      state = next
      onChange?.({ newState: next, oldState: prev })
      for (const listener of listeners) listener()
    },

    subscribe: (listener: Listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}
```

## 4. 状态变更处理

### 4.1 onChangeAppState

**文件**: `src/state/onChangeAppState.ts`

`onChangeAppState` 是中央状态变更处理器，负责副作用和持久化：

```typescript
export function onChangeAppState({
  newState,
  oldState,
}: {
  newState: AppState
  oldState: AppState
}) {
  // 1. 权限模式变更同步
  const prevMode = oldState.toolPermissionContext.mode
  const newMode = newState.toolPermissionContext.mode
  if (prevMode !== newMode) {
    const prevExternal = toExternalPermissionMode(prevMode)
    const newExternal = toExternalPermissionMode(newMode)
    if (prevExternal !== newExternal) {
      const isUltraplan =
        newExternal === 'plan' &&
        newState.isUltraplanMode &&
        !oldState.isUltraplanMode
          ? true
          : null
      notifySessionMetadataChanged({
        permission_mode: newExternal,
        is_ultraplan_mode: isUltraplan,
      })
    }
    notifyPermissionModeChanged(newMode)
  }

  // 2. 模型设置持久化
  if (newState.mainLoopModel !== oldState.mainLoopModel) {
    if (newState.mainLoopModel === null) {
      updateSettingsForSource('userSettings', { model: undefined })
      setMainLoopModelOverride(null)
    } else {
      updateSettingsForSource('userSettings', { model: newState.mainLoopModel })
      setMainLoopModelOverride(newState.mainLoopModel)
    }
  }

  // 3. 视图扩展持久化
  if (newState.expandedView !== oldState.expandedView) {
    const showExpandedTodos = newState.expandedView === 'tasks'
    const showSpinnerTree = newState.expandedView === 'teammates'
    if (
      getGlobalConfig().showExpandedTodos !== showExpandedTodos ||
      getGlobalConfig().showSpinnerTree !== showSpinnerTree
    ) {
      saveGlobalConfig(current => ({
        ...current,
        showExpandedTodos,
        showSpinnerTree,
      }))
    }
  }

  // 4. 详细模式持久化
  if (newState.verbose !== oldState.verbose) {
    saveGlobalConfig(current => ({
      ...current,
      verbose: newState.verbose,
    }))
  }

  // 5. 设置变更时清除认证缓存
  if (newState.settings !== oldState.settings) {
    clearApiKeyHelperCache()
    clearAwsCredentialsCache()
    clearGcpCredentialsCache()
    
    if (newState.settings.env !== oldState.settings.env) {
      applyConfigEnvironmentVariables()
    }
  }
}
```

```typescript
// src/components/hooks/useAppState.ts

export function useAppState<T>(
  selector: (state: AppState) => T,
  equalityFn?: (a: T, b: T) => boolean
): T {
  const store = useContext(AppStateContext)
  const [state, setState] = useState(() => selector(store.getState()))
  
  useEffect(() => {
    return store.subscribe((newState) => {
      const newValue = selector(newState)
      if (!equalityFn || !equalityFn(state, newValue)) {
        setState(newValue)
      }
    })
  }, [store, selector, equalityFn])
  
  return state
}

// 使用示例 - 细粒度订阅
const taskStatus = useAppState(
  state => state.tasks[taskId]?.status,
  (a, b) => a === b  // 仅当状态变化时重渲染
)
```

## 8. 总结

Claude Code 的状态管理系统展现了复杂应用的架构最佳实践：

### 架构优势

| 方面 | 实现 | 收益 |
|------|------|------|
| 分层设计 | 全局状态 + 应用状态 + 上下文 | 关注点分离，避免状态膨胀 |
| 类型安全 | 完整 TypeScript 类型 | 编译期错误检测 |
| 性能优化 | 选择器订阅 + 稳定引用 | 最小化重渲染 |
| 持久化 | 多层存储策略 | 数据安全，快速恢复 |
| 副作用管理 | 集中式 onChange 处理器 | 可维护，可测试 |

### 关键设计决策

1. **全局状态最小化**: 仅存储跨模块必需的状态
2. **访问器模式**: 通过 getter/setter 控制全局状态修改
3. **不可变更新**: 应用状态使用不可变更新模式
4. **细粒度订阅**: 选择器模式避免过度渲染
5. **批量处理**: 交互时间等高频更新批量处理
