# CM-03 - 历史记录与持久化

## 1. 概述

历史记录系统负责记录用户的命令输入，支持跨会话共享和检索。系统设计目标：

1. **跨会话历史**：不同会话之间共享历史
2. **高性能读写**：使用 JSONL 格式支持流式追加
3. **大内容外部存储**：粘贴内容 hash-based 外部存储
4. **双缓冲机制**：内存 pending + 磁盘持久化
5. **可撤销性**：支持 auto-restore-on-interrupt 场景

---

## 2. 存储格式

### 2.1 JSONL 格式

```typescript
type LogEntry = {
  display: string                    // 用户输入的显示文本
  pastedContents: Record<number, StoredPastedContent>  // 粘贴内容引用
  timestamp: number                  // 时间戳
  project: string                    // 项目路径
  sessionId?: string                 // 会话 ID
}
```

**存储格式**（history.jsonl）：
```json
{"display":"ls -la","pastedContents":{},"timestamp":1712345678901,"project":"/home/user/myapp","sessionId":"abc123"}
{"display":"npm install","pastedContents":{},"timestamp":1712345679012,"project":"/home/user/myapp","sessionId":"abc123"}
{"display":"git status","pastedContents":{},"timestamp":1712345680123,"project":"/home/user/myapp","sessionId":"def456"}
```

### 2.2 JSONL 格式的优势

| 优势 | 描述 |
|------|------|
| **增量写入** | 每条消息独立成行，支持流式追加，无需读取整个文件 |
| **容错性** | 单行损坏不影响其他行的解析 |
| **易解析** | 每行是完整的 JSON 对象，便于调试和手动处理 |
| **Tail 友好** | 可以用 `tail -n 100` 快速查看最近 100 条记录 |
| **并发安全** | 配合文件锁，多进程追加安全 |

## 3. 双缓冲机制

### 3.1 设计

```
┌─────────────────────────────────────────────────────────┐
│                    双缓冲架构                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  内存缓冲 (pendingEntries)                              │
│  ┌─────────────────────────────────────────┐           │
│  │ [entry1] [entry2] [entry3] ...          │           │
│  └─────────────────────────────────────────┘           │
│                    │                                    │
│                    │ flush (异步)                        │
│                    ▼                                    │
│  磁盘文件 (history.jsonl)                               │
│  ┌─────────────────────────────────────────┐           │
│  │ line1\n                                 │           │
│  │ line2\n                                 │           │
│  │ line3\n                                 │           │
│  │ ...                                     │           │
│  └─────────────────────────────────────────┘           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 实现

```typescript
let pendingEntries: LogEntry[] = []  // 内存缓冲
let isWriting = false                 // 写锁
let currentFlushPromise: Promise<void> | null = null
let lastAddedEntry: LogEntry | null = null  // 用于撤销
const skippedTimestamps = new Set<number>()  // 已写入但需跳过的时间戳

// 添加到缓冲
async function addToPromptHistory(command: HistoryEntry | string): Promise<void> {
  const entry = typeof command === 'string'
    ? { display: command, pastedContents: {} }
    : command
  
  // 处理粘贴内容（见第 4 节）
  const storedPastedContents = processPastedContents(entry.pastedContents)
  
  const logEntry: LogEntry = {
    ...entry,
    pastedContents: storedPastedContents,
    timestamp: Date.now(),
    project: getProjectRoot(),
    sessionId: getSessionId(),
  }
  
  pendingEntries.push(logEntry)
  lastAddedEntry = logEntry
  
  // 异步刷新
  void flushPromptHistory(0)
}

// 刷新逻辑
async function flushPromptHistory(retries: number): Promise<void> {
  if (isWriting || pendingEntries.length === 0) return
  
  if (retries > 5) return  // 失败超过 5 次放弃
  
  isWriting = true
  try {
    await immediateFlushHistory()
  } finally {
    isWriting = false
    if (pendingEntries.length > 0) {
      await sleep(500)
      void flushPromptHistory(retries + 1)
    }
  }
}

async function immediateFlushHistory(): Promise<void> {
  if (pendingEntries.length === 0) return
  
  const historyPath = join(getClaudeConfigHomeDir(), 'history.jsonl')
  
  // 确保文件存在
  await writeFile(historyPath, '', {
    encoding: 'utf8',
    mode: 0o600,  // 仅用户可读写
    flag: 'a',
  })
  
  // 获取文件锁
  let release
  try {
    release = await lock(historyPath, {
      stale: 10000,
      retries: { retries: 3, minTimeout: 50 },
    })
    
    const jsonLines = pendingEntries.map(entry => jsonStringify(entry) + '\n')
    pendingEntries = []  // 清空缓冲
    
    await appendFile(historyPath, jsonLines.join(''), { mode: 0o600 })
  } catch (error) {
    logForDebugging(`Failed to write prompt history: ${error}`)
  } finally {
    if (release) await release()
  }
}
```

### 3.3 读取流程

```typescript
async function* makeLogEntryReader(): AsyncGenerator<LogEntry> {
  const currentSession = getSessionId()
  
  // 1. 先从内存缓冲读取（最新的）
  for (let i = pendingEntries.length - 1; i >= 0; i--) {
    yield pendingEntries[i]!
  }
  
  // 2. 从磁盘读取（反向遍历，最新的在前）
  const historyPath = join(getClaudeConfigHomeDir(), 'history.jsonl')
  try {
    for await (const line of readLinesReverse(historyPath)) {
      try {
        const entry = deserializeLogEntry(line)
        
        // 跳过已撤销的条目
        if (
          entry.sessionId === currentSession &&
          skippedTimestamps.has(entry.timestamp)
        ) {
          continue
        }
        
        yield entry
      } catch (error) {
        logForDebugging(`Failed to parse history line: ${error}`)
      }
    }
  } catch (e: unknown) {
    const code = getErrnoCode(e)
    if (code === 'ENOENT') return  // 文件不存在，正常返回
    throw e
  }
}
```

## 4. 粘贴内容外部存储

### 4.1 问题背景

用户输入可能包含大段粘贴内容（文件内容、长文本）。如果直接存储在历史记录中：
- 历史记录文件迅速膨胀
- 跨会话同步效率低下
- 内存占用过大

### 4.2 解决方案：Hash-based 外部存储

```
┌─────────────────────────────────────────────────────────┐
│              粘贴内容存储架构                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  用户输入                                               │
│  ┌─────────────────────────────────────────┐           │
│  │ "console.log('hello')..." (5KB)         │           │
│  └─────────────────────────────────────────┘           │
│                    │                                    │
│                    ▼                                    │
│  ┌─────────────────────────────────────────┐           │
│  │ 大小判断                               │           │
│  │ if (size <= 1024) → inline             │           │
│  │ else → hash + external                 │           │
│  └─────────────────────────────────────────┘           │
│           │                    │                        │
│           ▼                    ▼                        │
│  历史记录中            粘贴存储中                        │
│  ┌─────────────────┐   ┌─────────────────┐            │
│  │ content: "..."  │   │ hash: abc123... │            │
│  │ (<1KB inline)   │   │ content: "..."  │            │
│  └─────────────────┘   │ (外部文件)      │            │
│                        └─────────────────┘            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.3 阈值与策略

```typescript
const MAX_PASTED_CONTENT_LENGTH = 1024  // 1KB 阈值

// StoredPastedContent 类型
type StoredPastedContent = {
  id: number
  type: 'text' | 'image'
  content?: string       // 小内容 inline
  contentHash?: string   // 大内容 hash 引用
  mediaType?: string
  filename?: string
}

// 处理逻辑
for (const [id, content] of Object.entries(entry.pastedContents)) {
  if (content.type === 'image') {
    continue  // 图像单独存储在 image-cache
  }
  
  if (content.content.length <= MAX_PASTED_CONTENT_LENGTH) {
    // 小内容：inline 存储
    storedPastedContents[Number(id)] = {
      id: content.id,
      type: content.type,
      content: content.content,
      mediaType: content.mediaType,
      filename: content.filename,
    }
  } else {
    // 大内容：hash 外部存储
    const hash = hashPastedText(content.content)
    storedPastedContents[Number(id)] = {
      id: content.id,
      type: content.type,
      contentHash: hash,
      mediaType: content.mediaType,
      filename: content.filename,
    }
    // 异步写入（fire-and-forget）
    void storePastedText(hash, content.content)
  }
}
```

### 4.4 读取时解析

```typescript
async function logEntryToHistoryEntry(entry: LogEntry): Promise<HistoryEntry> {
  const pastedContents: Record<number, PastedContent> = {}
  
  for (const [id, stored] of Object.entries(entry.pastedContents || {})) {
    const resolved = await resolveStoredPastedContent(stored)
    if (resolved) {
      pastedContents[Number(id)] = resolved
    }
  }
  
  return {
    display: entry.display,
    pastedContents,
  }
}

async function resolveStoredPastedContent(
  stored: StoredPastedContent,
): Promise<PastedContent | null> {
  // Inline content：直接使用
  if (stored.content) {
    return {
      id: stored.id,
      type: stored.type,
      content: stored.content,
      mediaType: stored.mediaType,
      filename: stored.filename,
    }
  }
  
  // Hash reference：从 paste store 获取
  if (stored.contentHash) {
    const content = await retrievePastedText(stored.contentHash)
    if (content) {
      return {
        id: stored.id,
        type: stored.type,
        content,
        mediaType: stored.mediaType,
        filename: stored.filename,
      }
    }
  }
  
  return null  // 内容不可用
}
```

### 4.5 引用格式

粘贴内容在 `display` 字段中的引用格式：

```
[Pasted text #1]           // 0 行（无换行）
[Pasted text #1 +10 lines] // 10 行
[Image #2]                 // 图像
[...Truncated text #3]     // 被截断的文本
```

```typescript
export function formatPastedTextRef(id: number, numLines: number): string {
  if (numLines === 0) {
    return `[Pasted text #${id}]`
  }
  return `[Pasted text #${id} +${numLines} lines]`
}

export function getPastedTextRefNumLines(text: string): number {
  return (text.match(/\r\n|\r|\n/g) || []).length
}
```

### 4.6 展开引用

```typescript
export function expandPastedTextRefs(
  input: string,
  pastedContents: Record<number, PastedContent>,
): string {
  const refs = parseReferences(input)
  let expanded = input
  
  // 反向替换（保持 offset 有效）
  for (let i = refs.length - 1; i >= 0; i--) {
    const ref = refs[i]!
    const content = pastedContents[ref.id]
    if (content?.type !== 'text') continue
    
    expanded =
      expanded.slice(0, ref.index) +
      content.content +
      expanded.slice(ref.index + ref.match.length)
  }
  
  return expanded
}

function parseReferences(
  input: string,
): Array<{ id: number; match: string; index: number }> {
  const referencePattern =
    /\[(Pasted text|Image|\.\.\.Truncated text) #(\d+)(?: \+\d+ lines)?(\.)*\]/g
  
  return [...input.matchAll(referencePattern)]
    .map(match => ({
      id: parseInt(match[2] || '0'),
      match: match[0],
      index: match.index!,
    }))
    .filter(match => match.id > 0)
}
```

## 5. 撤销机制（Auto-restore-on-interrupt）

### 5.1 场景

用户按下 Esc 中断当前提交时：
1. 对话在收到响应前被回滚
2. 语义上这次提交被撤销
3. 历史记录也应该撤销，否则 Up-arrow 会显示重复内容

### 5.2 实现

```typescript
// 快路径：在 flush 之前从 pending 缓冲移除
export function removeLastFromHistory(): void {
  if (!lastAddedEntry) return
  
  const entry = lastAddedEntry
  lastAddedEntry = null  // 一Shot：清空追踪
  
  const idx = pendingEntries.lastIndexOf(entry)
  if (idx !== -1) {
    // 还在内存缓冲中：直接移除
    pendingEntries.splice(idx, 1)
  } else {
    // 已写入磁盘：添加到 skip 集合
    skippedTimestamps.add(entry.timestamp)
  }
}
```

### 5.3  race condition 处理

```
时间线：
  t0: 用户提交 → addToHistory → entry 进入 pending
  t1: flush 开始 → 写入磁盘 → pending 清空
  t2: 用户按 Esc → removeLastFromHistory

在 t2 时：
  - 如果 entry 还在 pending：直接 splice 移除
  - 如果 entry 已写入磁盘：添加到 skippedTimestamps
    读取时会跳过这个 timestamp 的条目
```

### 5.4 Session 作用域

```typescript
// Session-scoped（模块状态在进程重启时重置）
const skippedTimestamps = new Set<number>()
```

**Rationale**：撤销只在当前会话中有意义，进程重启后不再需要跳过。

## 6. 历史读取优化

### 6.1 当前项目优先

```typescript
export async function* getHistory(): AsyncGenerator<HistoryEntry> {
  const currentProject = getProjectRoot()
  const currentSession = getSessionId()
  const otherSessionEntries: LogEntry[] = []
  let yielded = 0
  
  for await (const entry of makeLogEntryReader()) {
    if (!entry || typeof entry.project !== 'string') continue
    if (entry.project !== currentProject) continue
    
    if (entry.sessionId === currentSession) {
      // 当前会话优先
      yield await logEntryToHistoryEntry(entry)
      yielded++
    } else {
      // 其他会话稍后返回
      otherSessionEntries.push(entry)
    }
    
    if (yielded + otherSessionEntries.length >= MAX_HISTORY_ITEMS) break
  }
  
  // 返回其他会话的条目
  for (const entry of otherSessionEntries) {
    if (yielded >= MAX_HISTORY_ITEMS) return
    yield await logEntryToHistoryEntry(entry)
    yielded++
  }
}
```

**效果**：Up-arrow 历史中，当前会话的条目在前，其他会话的在后，不会交错。

### 6.2 带时间戳的历史（用于 ctrl+r picker）

```typescript
export type TimestampedHistoryEntry = {
  display: string
  timestamp: number
  resolve: () => Promise<HistoryEntry>
}

export async function* getTimestampedHistory(): AsyncGenerator<TimestampedHistoryEntry> {
  const currentProject = getProjectRoot()
  const seen = new Set<string>()
  
  for await (const entry of makeLogEntryReader()) {
    if (!entry || typeof entry.project !== 'string') continue
    if (entry.project !== currentProject) continue
    if (seen.has(entry.display)) continue  // 去重
    
    seen.add(entry.display)
    
    yield {
      display: entry.display,
      timestamp: entry.timestamp,
      resolve: () => logEntryToHistoryEntry(entry),
    }
    
    if (seen.size >= MAX_HISTORY_ITEMS) return
  }
}
```

**Lazy Resolve**：粘贴内容在 `resolve()` 时解析，picker 只需要 `display` 和 `timestamp` 来渲染列表。

## 7. 清理与限流

### 7.1 最大条目数

```typescript
const MAX_HISTORY_ITEMS = 100
```

遍历到 100 条后停止，防止读取过多历史。

### 7.2 清理注册

```typescript
let cleanupRegistered = false

export function addToHistory(command: HistoryEntry | string): void {
  if (isEnvTruthy(process.env.CLAUDE_CODE_SKIP_PROMPT_HISTORY)) {
    return  // Tmux  spawned 会话跳过
  }
  
  if (!cleanupRegistered) {
    cleanupRegistered = true
    registerCleanup(async () => {
      // 等待进行中的 flush
      if (currentFlushPromise) {
        await currentFlushPromise
      }
      // 最终 flush
      if (pendingEntries.length > 0) {
        await immediateFlushHistory()
      }
    })
  }
  
  void addToPromptHistory(command)
}
```

  const code = getErrnoCode(e)

## 10. 总结

历史记录系统的设计要点：

| 机制 | 描述 |
|------|------|
| JSONL 格式 | 增量写入、容错、易解析 |
| 双缓冲 | pendingEntries（内存）+ history.jsonl（磁盘） |
| Hash 外部存储 | >1KB 内容 hash-based 外部存储 |
| 撤销机制 | removeLastFromHistory 支持 auto-restore-on-interrupt |
| Session 优先 | 当前会话条目优先返回 |
| Lazy resolve | 粘贴内容延迟解析 |
| 文件锁 | 并发安全的追加写入 |
| 权限控制 | 0o600 仅用户可读写 |

这套系统在保证性能的同时，提供了灵活的历史管理和可靠的持久化能力。
