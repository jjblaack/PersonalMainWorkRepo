### 3.4 查询引擎 (QueryEngine.ts)

**位置**: `src/QueryEngine.ts`
**行数**: ~1,295 行

**核心职责**:
1. LLM API 调用管理
2. 流式响应处理
3. Tool Call 循环
4. Token 计数
5. 重试逻辑
6. 错误处理

**关键类结构**:

```typescript
class QueryEngine {
  // 配置
  private model: string;
  private maxTokens: number;
  private systemPrompt: string;
  
  // 状态
  private conversation: Message[];
  private tokenCount: TokenCount;
  
  // 核心方法
  async send(message: Message): Promise<Response> {
    // 1. 构建 API 请求
    const request = this.buildRequest(message);
    
    // 2. 发送 API 请求
    const response = await this.api.messages.create(request);
    
    // 3. 处理流式响应
    for await (const chunk of response.stream) {
      this.handleChunk(chunk);
    }
    
    // 4. 处理 Tool Call
    if (response.hasToolCalls) {
      return this.processToolCalls(response);
    }
    
    return response;
  }
  
  private async processToolCalls(response: Response) {
    // 执行工具调用循环
    while (response.hasToolCalls) {
      const results = await this.executeTools(response.toolCalls);
      response = await this.sendToolResults(results);
    }
    return response;
  }
}
```

### 4.2 核心依赖链

```
用户输入
```

### 4.3 循环依赖避免

项目使用以下策略避免循环依赖:

1. **分层架构**: 严格的层次依赖关系
2. **接口隔离**: 使用接口而非具体实现
3. **依赖注入**: 通过参数传递依赖
4. **懒加载**: 使用动态 import 打破循环

## 6. 文件命名规范

### 6.1 通用命名规则

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| TypeScript 源文件 | PascalCase 或 camelCase | `QueryEngine.ts`, `messages.js` |
| React 组件 | PascalCase + .tsx | `App.tsx`, `FullscreenLayout.tsx` |
| 工具实现 | PascalCase + Tool | `BashTool.ts`, `FileReadTool.ts` |
| 命令实现 | camelCase | `commit.ts`, `review.ts` |
| 类型定义 | camelCase | `message.ts`, `ids.ts` |
| 测试文件 | *.test.ts 或 *.spec.ts | `utils.test.ts` |

### 6.2 目录命名规则

| 目录类型 | 命名规则 | 示例 |
|----------|----------|------|
| 功能模块 | 复数名词 | `commands/`, `tools/`, `components/` |
| 服务层 | 单数/复数混合 | `api/`, `mcp/`, `analytics/` |
| 工具函数 | 复数名词 | `utils/`, `hooks/` |

### 6.3 导出模式

```typescript
// 单个导出 (推荐)
export class FileReadTool extends Tool { ... }

// 批量导出
export { FileReadTool } from './FileReadTool.js';
export { FileWriteTool } from './FileWriteTool.js';

// 默认导出 (较少使用)
export default function helper() { ... }
```

## 总结

Claude Code 的目录结构体现了以下设计原则：

1. **关注点分离**: 命令、工具、服务、UI 各司其职
2. **模块化**: 每个工具/命令都是独立的模块
3. **层次清晰**: 入口→命令→工具→服务→基础设施
4. **可扩展**: 新增工具/命令不影响现有代码
5. **类型安全**: TypeScript 严格类型贯穿整个项目

这种组织结构使得近 2000 个文件的大型项目依然保持良好的可维护性和可读性。
