## 1. 项目背景

### 1.1 源代码暴露事件

2026 年 3 月 31 日，Chaofan Shou (@Fried_rice) 在 Twitter 上公开指出，Claude Code 的源代码通过 npm 包中暴露的 source map 文件可以被访问：

> "Claude code source code has been leaked via a map file in their npm registry!"
> — [@Fried_rice, March 31, 2026](https://x.com/Fried_rice/status/2038894956459290963)

该 source map 引用了存储在 Anthropic R2 存储桶中的未混淆 TypeScript 源代码，使得整个 `src/` 目录的快照可以被公开下载。

### 1.2 仓库建立目的

本分析项目基于一个**公开可用的 Claude Code 源代码快照**，该快照于 2026 年 3 月 31 日通过 npm 分发中的 source map 暴露而变得可访问。

此仓库由一名大学生维护，用于：
- 软件供应链暴露和构建产物泄露研究
- 安全软件工程实践学习
- 智能体开发者工具架构分析
- 真实世界 CLI 系统的防御性分析

### 1.3 项目规模

- **语言**: TypeScript (严格模式)
- **运行时**: Bun
- **终端 UI**: React + Ink
- **代码规模**: ~1,900 个文件，512,000+ 行代码

## 2. 项目定位

### 2.1 什么是 Claude Code？

Claude Code 是 Anthropic 官方开发的 CLI 工具，允许用户通过终端与 Claude AI 进行交互，执行各种软件工程任务。

### 2.2 核心功能

- **文件操作**: 读取、写入、编辑代码文件
- **代码搜索**: 使用 ripgrep 进行内容搜索，使用 glob 进行文件匹配
- **命令执行**: 运行 Shell 命令
- **代码审查**: 分析代码并提供改进建议
- **Git 集成**: 创建 commit、查看 diff
- **多代理协作**:  spawned 子代理并行工作
- **MCP 集成**: Model Context Protocol 服务器支持
- **LSP 集成**: 语言服务器协议支持
- **技能系统**: 可复用工作流定义
- **插件系统**: 第三方扩展支持

### 2.3 使用场景

```bash
# 基本使用
claude "帮我重构这个函数"

# 带文件上下文
claude "分析 src/auth.ts 的安全问题"

# 使用命令
claude /review
claude /commit
claude /mcp
```

## 6. 核心设计理念

### 6.1 工具系统设计哲学

**核心原则**: 每个工具都是自包含的模块，定义自己的输入 Schema、权限模型和执行逻辑。

```typescript
// 工具接口抽象 (简化)
interface Tool {
  name: string;
  description: string;
  inputSchema: ToolInputJSONSchema;
  execute(input: any, context: ToolContext): Promise<ToolResult>;
}
```

**设计优势**:
- **模块化**: 每个工具独立开发和测试
- **可扩展**: 添加新工具不影响现有工具
- **类型安全**: Schema 验证确保输入正确
- **权限隔离**: 每个工具定义自己的权限需求

### 6.2 命令与工具的关系

**命令 (Commands)**: 用户通过 `/` 前缀调用的面向用户的接口
**工具 (Tools)**: AI 模型可以调用的底层能力

```
用户输入: "帮我检查这个文件"
         │
         ▼
    ┌─────────────┐
    │  AI 理解意图  │
    └─────────────┘
         │
         ▼
    ┌─────────────┐
    │ 调用 FileReadTool │  ← 工具是 AI 的能力
    └─────────────┘

用户输入: /review
         │
         ▼
    ┌─────────────┐
    │ 命令处理器   │  ← 命令是用户的快捷方式
    └─────────────┘
         │
         ▼
    ┌─────────────┐
    │ 调用多个工具 │
    └─────────────┘
```

### 6.3 多代理架构思想

Claude Code 支持 spawned 多个子代理并行工作:

```
主代理 (Main Agent)
    │
    ├───► 子代理 A (AgentTool) ────► 任务 A
    │
    ├───► 子代理 B (AgentTool) ────► 任务 B
    │
    └───► 子代理 C (AgentTool) ────► 任务 C
    
结果聚合 ◄──────────────────────────┘
```

**协调器模式** (`coordinator/`):
- 任务分解与分配
- 代理间通信
- 结果聚合与展示

### 6.4 权限模型

**权限级别**:
1. **低权限**: 读取文件、搜索代码
2. **中权限**: 编辑文件、运行安全命令
3. **高权限**: 删除文件、执行任意命令

**权限模式**:
- `default` - 默认模式，需要审批
- `plan` - 计划模式，批量审批
- `auto` - 自动模式，低风险工具自动执行
- `bypassPermissions` - 开发模式，绕过审批

**审批流程**:
```
工具调用请求
    │
    ▼
权限检查 (hooks/toolPermission/)
    │
    ├── 已授权 ──► 直接执行
    │
    └── 未授权 ──► 用户审批对话框
                     │
                     ├── 批准 ──► 执行 + 记录
                     │
                     └── 拒绝 ──► 返回错误
```

### 6.5 性能优化模式

**并行预取**:
```typescript
// main.tsx 启动时的并行预取
startMdmRawRead()      // MDM 设置读取
startKeychainPrefetch() // Keychain 认证预取
// 与剩余 import 并行执行
```

**惰性加载**:
```typescript
// 仅在需要时加载
const coordinatorModule = feature('COORDINATOR_MODE') 
  ? require('./coordinator/coordinatorMode.js')
  : null;
```

**特性标志死代码消除**:
```typescript
import { feature } from 'bun:bundle';

// 构建时消除未启用功能的代码
const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null;
```
