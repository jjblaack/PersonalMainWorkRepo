> 基于 Claude Code 项目经验，总结构建 24 小时超级 Agent 需要避免的陷阱。

## 1. 状态管理相关的坑

### 1.1 状态未持久化

**问题**：关键状态只存在内存中，进程崩溃后丢失。

**表现**：
- 重启后任务进度丢失
- 记忆丢失
- 需要从头开始

**解决方案**：
```typescript
// ❌ 错误：状态只在内存中
class TaskManager {
  private tasks: Map<string, Task> = new Map()  // 崩溃后丢失
}

// ✅ 正确：状态持久化
class TaskManager {
  private tasks: Map<string, Task> = new Map()
  private stateDir: string
  
  async createTask(spec: TaskSpec): Promise<Task> {
    const task = { ... }
    this.tasks.set(task.id, task)
    
    // 立即持久化
    await this.persistTask(task)
    
    return task
  }
  
  async updateTaskStatus(taskId: string, status: TaskStatus): Promise<void> {
    const task = this.tasks.get(taskId)!
    task.status = status
    this.tasks.set(taskId, task)
    
    // 立即持久化
    await this.persistTask(task)
  }
  
  private async persistTask(task: Task): Promise<void> {
    const statePath = join(this.stateDir, `${task.id}.json`)
    const tempPath = statePath + '.tmp'
    
    // 写入临时文件
    await write(tempPath, JSON.stringify(task))
    
    // 原子替换
    await rename(tempPath, statePath)
  }
}
```

### 1.2 状态更新不同步

**问题**：多个地方修改同一状态，导致不一致。

**表现**：
- 任务状态与实际执行不符
- 工人状态与实际不符
- 数据竞争

**解决方案**：
```typescript
// ❌ 错误：分散更新
task.status = 'completed'
saveTask(task)
notifyUser(task)  // 如果这里失败，用户不会收到通知

// ✅ 正确：使用单一更新函数
class StateManager {
  updateTask(taskId: string, updater: (task: Task) => Task): void {
    const task = this.tasks.get(taskId)!
    const updated = updater(task)
    
    // 事务性更新
    this.tasks.set(taskId, updated)
    this.persistTask(updated)
    this.emitEvent('task:updated', updated)
  }
}

// 使用
stateManager.updateTask(taskId, task => ({
  ...task,
  status: 'completed',
  endTime: Date.now(),
}))
```

### 1.3 忘记处理终端状态

**问题**：任务进入终端状态 (completed/failed) 后还能被修改。

**表现**：
- 已完成的任务被重新启动
- 状态转换混乱

**解决方案**：
```typescript
// ✅ 正确：检查终端状态
const TERMINAL_STATUSES = ['completed', 'failed', 'killed', 'deleted']

function isTerminalStatus(status: TaskStatus): boolean {
  return TERMINAL_STATUSES.includes(status)
}

function updateTaskStatus(task: Task, newStatus: TaskStatus): Task {
  // 检查是否可以从当前状态转换
  if (isTerminalStatus(task.status) && task.status !== newStatus) {
    throw new Error(
      `Cannot transition from terminal status ${task.status} to ${newStatus}`
    )
  }
  
  // 检查是否是有效的转换
  if (!isValidTransition(task.status, newStatus)) {
    throw new Error(`Invalid transition from ${task.status} to ${newStatus}`)
  }
  
  return { ...task, status: newStatus, updatedAt: Date.now() }
}
```

## 2. 任务管理相关的坑

### 2.1 循环依赖

**问题**：任务 A 依赖任务 B，任务 B 依赖任务 A，形成死锁。

**表现**：
- 任务永远无法开始
- 系统挂起

**检测方案**：
```typescript
// ✅ 创建任务时检测循环依赖
function hasCircularDependency(
  taskId: string,
  tasks: Map<string, Task>,
  visited = new Set<string>(),
  stack = new Set<string>()
): boolean {
  if (stack.has(taskId)) return true  // 检测到循环
  if (visited.has(taskId)) return false
  
  visited.add(taskId)
  stack.add(taskId)
  
  const task = tasks.get(taskId)
  for (const depId of task?.blockedBy || []) {
    if (hasCircularDependency(depId, tasks, visited, stack)) {
      return true
    }
  }
  
  stack.delete(taskId)
  return false
}

// 创建任务时检查
async function createTaskWithDeps(spec: TaskSpec): Promise<Task> {
  // 临时添加依赖
  const tempTask = { ...spec, blockedBy: spec.blockedBy }
  
  // 检测循环
  if (hasCircularDependency(spec.id, tasks)) {
    throw new Error('Circular dependency detected')
  }
  
  return createTask(spec)
}
```

### 2.2 任务超时未处理

**问题**：任务执行时间超过预期，没有超时处理。

**表现**：
- 任务永远"进行中"
- 工人资源被占用

**解决方案**：
```typescript
// ✅ 设置超时
class TaskExecutor {
  private timeouts: Map<string, NodeJS.Timeout> = new Map()
  
  async executeWithTimeout(taskId: string, timeoutMs: number): Promise<void> {
    // 设置超时定时器
    const timeout = setTimeout(async () => {
      await this.handleTimeout(taskId)
    }, timeoutMs)
    
    this.timeouts.set(taskId, timeout)
    
    try {
      await this.execute(taskId)
    } finally {
      clearTimeout(timeout)
      this.timeouts.delete(taskId)
    }
  }
  
  private async handleTimeout(taskId: string): Promise<void> {
    logger.warn(`Task ${taskId} timed out`)
    
    // 停止工人
    const task = this.tasks.get(taskId)!
    if (task.owner) {
      await this.workerPool.killWorker(task.owner)
    }
    
    // 标记为失败
    await this.taskManager.failTask(taskId, 'Task timed out')
  }
}
```

### 2.3 失败任务无恢复

**问题**：任务失败后没有重试或恢复机制。

**表现**：
- 临时错误导致任务永久失败
- 需要人工干预

**解决方案**：
```typescript
// ✅ 自动重试
class TaskRecovery {
  private readonly MAX_RETRIES = 3
  
  async executeWithRetry(
    taskId: string,
    fn: () => Promise<void>
  ): Promise<void> {
    let lastError: Error
    
    for (let i = 0; i < this.MAX_RETRIES; i++) {
      try {
        return await fn()
      } catch (error) {
        lastError = error as Error
        
        // 检查是否可重试
        if (!this.isRetryable(error)) {
          throw error  // 不可重试的错误直接抛出
        }
        
        // 指数退避
        const delay = Math.pow(2, i) * 1000
        logger.info(`Retry ${i + 1} after ${delay}ms`)
        await sleep(delay)
      }
    }
    
    // 所有重试失败
    await this.taskManager.failTask(taskId, lastError!.message)
  }
  
  private isRetryable(error: unknown): boolean {
    // 网络错误、超时等可重试
    // 验证错误、权限错误等不可重试
    return error instanceof NetworkError || 
           error instanceof TimeoutError
  }
}
```

## 3. 记忆管理相关的坑

### 3.1 保存过多无用记忆

**问题**：把可以从代码推导的信息也保存为记忆。

**表现**：
- 记忆文件膨胀
- 检索变慢
- 有用信息被淹没

**解决方案**：
```typescript
// ✅ 过滤无用记忆
function shouldSaveMemory(memory: PotentialMemory): boolean {
  // 不应该保存的情况
  if (isDerivableFromCode(memory)) {
    logger.debug(`Skipping memory: derivable from code`)
    return false
  }
  
  if (isInGitHistory(memory)) {
    logger.debug(`Skipping memory: in git history`)
    return false
  }
  
  if (isEphemeral(memory)) {
    logger.debug(`Skipping memory: ephemeral`)
    return false
  }
  
  // 应该保存的情况
  if (memory.type === 'feedback') return true
  if (memory.type === 'user' && isNonObvious(memory)) return true
  if (memory.type === 'project' && hasLongTermValue(memory)) return true
  if (memory.type === 'reference') return true
  
  return false
}

// 检查是否可从代码推导
function isDerivableFromCode(memory: PotentialMemory): boolean {
  // 代码模式、函数签名、文件结构等都可从代码推导
  const derivablePatterns = [
    /function.*\(/,
    /class\s+\w+/,
    /interface\s+\w+/,
    /import.*from/,
  ]
  
  return derivablePatterns.some(pattern => 
    pattern.test(memory.content)
  )
}
```

### 3.2 记忆索引过大

**问题**：MEMORY.md 索引文件过大，超出加载限制。

**表现**：
- 索引被截断
- 部分记忆无法加载

**Claude Code 的限制**：
```typescript
const MAX_ENTRYPOINT_LINES = 200
const MAX_ENTRYPOINT_BYTES = 25_000  // ~25KB
```

**解决方案**：
```typescript
// ✅ 索引条目简洁
function formatIndexEntry(memory: Memory): string {
  // 简洁的描述，包含关键信息
  const shortDesc = memory.description.slice(0, 50)
  return `- [${memory.name}](${memory.fileName}.md) — ${shortDesc}`
}

// ✅ 定期清理索引
async function cleanupIndex(): Promise<void> {
  const index = await loadIndex()
  
  // 移除已删除的记忆
  const existingFiles = await readdir(MEMORY_DIR)
  const cleaned = index.entries.filter(
    entry => existingFiles.includes(entry.fileName + '.md')
  )
  
  // 检查是否超出限制
  const content = formatIndex({ entries: cleaned })
  
  if (content.length > MAX_ENTRYPOINT_BYTES) {
    // 移除最旧的条目
    cleaned.sort((a, b) => a.createdAt - b.createdAt)
    while (content.length > MAX_ENTRYPOINT_BYTES) {
      cleaned.shift()
    }
  }
  
  await saveIndex({ entries: cleaned })
}
```

### 3.3 记忆注入时机不当

**问题**：在错误的时机注入记忆，导致上下文混乱。

**表现**：
- 记忆干扰当前任务
- 不相关的记忆被激活

**解决方案**：
```typescript
// ✅ 按场景注入记忆
class MemoryInjector {
  // 启动时：注入所有记忆
  async injectAtStartup(): Promise<string> {
    const memories = await this.loadAllMemories()
    return this.formatMemories(memories)
  }
  
  // 创建任务时：注入相关记忆
  async injectForTask(task: Task): Promise<string> {
    // 检索与任务相关的记忆
    const query = `${task.subject} ${task.description}`
    const relevant = await this.retrieveMemories(query)
    
    return this.formatMemories(relevant)
  }
  
  // 执行中：不主动注入，除非显式查询
  async queryMemory(query: string): Promise<string> {
    const memories = await this.retrieveMemories(query)
    return this.formatMemories(memories)
  }
  
  private formatMemories(memories: Memory[]): string {
    const sections = {
      user: '## User Context\n',
      feedback: '## Guidelines\n',
      project: '## Project Context\n',
      reference: '## References\n',
    }
    
    for (const memory of memories) {
      sections[memory.type] += `- ${memory.content}\n`
    }
    
    return Object.values(sections)
      .filter(s => s.trim().length > 0)
      .join('\n\n')
  }
}
```

## 4. 工人管理相关的坑

### 4.1 工人泄漏

**问题**：工人创建后没有正确释放，导致资源耗尽。

**表现**：
- 可用工人越来越少
- 最终无法创建新任务

**解决方案**：
```typescript
// ✅ 正确管理工人生命周期
class WorkerPool {
  private workers: Map<string, Worker> = new Map()
  
  async acquireWorker(): Promise<Worker> {
    // 查找空闲工人
    for (const worker of this.workers.values()) {
      if (worker.status === 'idle') {
        worker.status = 'busy'
        return worker
      }
    }
    
    // 创建新工人
    const worker = await this.createWorker()
    this.workers.set(worker.id, worker)
    return worker
  }
  
  async releaseWorker(workerId: string): Promise<void> {
    const worker = this.workers.get(workerId)
    if (worker) {
      worker.status = 'idle'
      worker.currentTask = undefined
      
      // 定期清理长时间空闲的工人
      this.scheduleIdleCleanup(workerId)
    }
  }
  
  private scheduleIdleCleanup(workerId: string): void {
    setTimeout(() => {
      const worker = this.workers.get(workerId)
      if (worker && worker.status === 'idle' && this.workers.size > this.minWorkers) {
        worker.cleanup()
        this.workers.delete(workerId)
        logger.info(`Cleaned up idle worker ${workerId}`)
      }
    }, IDLE_TIMEOUT_MS)
  }
  
  async shutdown(): Promise<void> {
    // 关闭所有工人
    for (const worker of this.workers.values()) {
      await worker.cleanup()
    }
    this.workers.clear()
  }
}
```

## 5. 24 小时运行相关的坑

### 5.1 没有心跳机制

**问题**：连接断开后无法及时发现。

**表现**：
- 服务器认为客户端已下线
- 会话过期

**解决方案**：
```typescript
// ✅ 定期发送心跳
class HeartbeatManager {
  private interval: NodeJS.Timeout | null = null
  private readonly INTERVAL_MS = 60000  // 60 秒
  private readonly TIMEOUT_MS = 300000  // 5 分钟
  
  start(sessionId: string, token: string): void {
    this.interval = setInterval(async () => {
      try {
        await this.sendHeartbeat(sessionId, token)
      } catch (error) {
        logger.error('Heartbeat failed:', error)
        this.triggerReconnect()
      }
    }, this.INTERVAL_MS)
  }
  
  private async sendHeartbeat(
    sessionId: string,
    token: string
  ): Promise<void> {
    const response = await fetch(
      `${API_BASE}/sessions/${sessionId}/heartbeat`,
      {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${token}` },
      }
    )
    
    if (!response.ok) {
      throw new Error(`Heartbeat failed: ${response.status}`)
    }
  }
  
  stop(): void {
    if (this.interval) {
      clearInterval(this.interval)
      this.interval = null
    }
  }
}
```

### 5.3 内存泄漏

**问题**：内存使用持续增长，最终 OOM。

**表现**：
- 内存使用只增不减
- 运行几天后崩溃

**检测和解决方案**：
```typescript
// ✅ 内存监控
class MemoryMonitor {
  private readonly LIMIT_MB = 1024
  
  startMonitoring(): void {
    setInterval(() => {
      const usage = process.memoryUsage()
      const heapUsedMB = usage.heapUsed / 1024 / 1024
      
      logger.debug(`Memory: ${heapUsedMB.toFixed(1)}MB / ${this.LIMIT_MB}MB`)
      
      if (heapUsedMB > this.LIMIT_MB * 0.8) {
        logger.warn('High memory usage, triggering GC')
        
        // 手动 GC (如果启用)
        if (global.gc) {
          global.gc()
        }
        
        // 清理缓存
        this.cleanupCaches()
      }
      
      if (heapUsedMB > this.LIMIT_MB * 0.95) {
        logger.error('Critical memory usage, restarting...')
        this.gracefulRestart()
      }
    }, 60000)  // 每分钟检查
  }
  
  private cleanupCaches(): void {
    // 清理文件缓存
    fileCache.clear()
    
    // 清理短期记忆
    shortTermMemory.trim(100)  // 保留最近 100 条
    
    // 清理已完成的任务状态
    this.cleanupCompletedTasks()
  }
  
  private gracefulRestart(): void {
    // 保存状态
    stateManager.saveState()
    
    // 退出进程 (由守护进程重启)
    process.exit(0)
  }
}
```

## 6. 并发和竞态条件

### 6.1 同时修改同一任务

**问题**：多个地方同时修改同一任务，导致数据损坏。

**解决方案**：
```typescript
// ✅ 使用乐观锁
class TaskManager {
  async updateTask(
    taskId: string,
    updater: (task: Task) => Task,
    expectedVersion: number
  ): Promise<Task> {
    const task = await this.loadTask(taskId)
    
    // 检查版本
    if (task.version !== expectedVersion) {
      throw new ConcurrentModificationError(
        `Task ${taskId} was modified by another process`
      )
    }
    
    const updated = updater(task)
    updated.version++
    
    return this.saveTask(updated)
  }
}
```
