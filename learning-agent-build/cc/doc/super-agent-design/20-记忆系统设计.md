# 记忆系统设计

> 分层记忆模型、记忆的提取、存储、检索和注入机制。

---

## 1. 记忆分层模型

### 1.1 三层记忆架构

```
┌─────────────────────────────────────────────────────────┐
│                    记忆分层模型                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  第一层：工作记忆 (Working Memory)                       │
│  ─────────────────────────────────                       │
│  容量：约 4±1 个"组块" ( Miller's Law)                     │
│  持久性：秒到分钟                                        │
│  内容：当前任务的上下文、临时变量                        │
│  类比：CPU 寄存器                                         │
│                                                         │
│  第二层：短期记忆 (Short-term Memory)                    │
│  ──────────────────────────────────                      │
│  容量：最近 N 轮对话 (可配置，如 50-100 轮)                  │
│  持久性：会话期间                                        │
│  内容：对话历史、最近的文件操作、最近的命令              │
│  类比：RAM                                               │
│                                                         │
│  第三层：长期记忆 (Long-term Memory)                     │
│  ───────────────────────────────────                     │
│  容量：无限制                                            │
│  持久性：永久                                            │
│  内容：用户偏好、反馈、项目背景、外部资源                │
│  类比：硬盘/SSD                                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 各层记忆的特点

| 特性 | 工作记忆 | 短期记忆 | 长期记忆 |
|------|----------|----------|----------|
| 容量 | 很小 (4±1) | 中等 (N 轮) | 无限 |
| 持久性 | 秒 - 分钟 | 会话期间 | 永久 |
| 访问速度 | 最快 | 快 | 中等 (需检索) |
| 存储位置 | 内存 | 内存 + 临时文件 | 磁盘文件 |
| 更新频率 | 极高 | 高 | 低 |
| 是否需要持久化 | 否 | 可选 | 是 |

## 2. 长期记忆的类型

### 2.1 四种记忆类型

基于 Claude Code 的记忆系统，超级 Agent 使用四种记忆类型：

#### User 记忆
记录用户的背景信息、角色、技术栈、偏好等。

```markdown
---
name: data_scientist_profile
description: 用户是数据科学家，专注于数据管道
type: user
---

用户是一名数据科学家，主要工作是构建和维护数据管道。
技术栈包括 Python、SQL、Apache Spark、Airflow。

**Why:** 用户在对话中明确说明了自己的角色和技术栈。
**How to apply:** 
- 解释概念时使用数据管道相关的类比
- 优先推荐数据处理相关的库和工具
- 代码示例使用 Python
```

#### Feedback 记忆
记录用户对 Agent 工作的反馈和指导。

```markdown
---
name: no_mock_in_tests
description: 测试使用真实数据库，不要用 mock
type: feedback
---

测试应该连接真实的 PostgreSQL 数据库，不要使用 mock 或 in-memory 数据库。

**Why:** 之前 mock 测试通过了但生产环境失败，因为 mock 没有捕捉到真实的数据库行为。
**How to apply:** 
- 编写新测试时使用 Docker 启动测试数据库
- 或者使用项目的 testcontainers 配置
- 不要在单元测试中使用 mock 数据库
```

#### Project 记忆
记录项目相关的背景、决策、正在进行的工作等。

```markdown
---
name: auth_refactor_compliance
description: 认证模块重构是法律合规驱动
type: project
---

正在进行的认证模块重构是为了满足新的 GDPR 合规要求，不是技术债务清理。

**Why:** 法律团队 flagged 当前 session token 存储方式不符合合规要求。
**How to apply:** 
- 所有认证相关决策应优先考虑合规性而非开发便利性
- 文档中需要说明变更的合规原因
- 测试需要覆盖合规场景
```

#### Reference 记忆
记录外部资源的位置和引用信息。

```markdown
---
name: linear_pipeline_bugs
description: 数据管道 bug 追踪在 Linear 项目 INGEST
type: reference
---

所有数据管道相关的 bug 都追踪在 Linear 项目的 "INGEST" 项目中。

**Why:** 团队选择将管道 bug 与主产品 bug 分开追踪。
**How to apply:** 
- 查询管道 bug 时使用 Linear 项目 INGEST
- 创建相关 ticket 时选择 INGEST 项目
```

### 2.2 什么应该保存为记忆

**应该保存**：
- 用户角色和背景信息
- 用户明确的工作偏好
- 从错误中学习的经验
- 非显而易见的架构决策
- 外部资源位置 (bug 追踪、仪表盘等)
- 团队约定的工作流
- 项目关键决策和背景

**不应该保存**：
- 代码模式和技术细节 (可从代码推导)
- Git 历史已有信息
- 临时任务状态
- 显而易见的偏好
- 文件内容本身 (应保存的是文件的意义)

## 3. 记忆的生命周期

### 3.1 记忆生命周期流程

```
创建 ──► 存储 ──► 检索 ──► 注入 ──► 使用 ──► 更新/过期
 │        │        │        │        │         │
 │        │        │        │        │         │
 │        │        │        │        │         └─► 发现记忆不准确
 │        │        │        │        │             或项目状态变化
 │        │        │        │        │
 │        │        │        │        └─► 每次对话自动加载
 │        │        │        │
 │        │        │        └─► 将记忆注入上下文
 │        │        │
 │        │        └─► 关键词/语义检索
 │        │
 │        └─► 写入文件 + 更新索引
 │
 └─► 用户明确请求或自动提取
```

### 3.2 记忆提取

```typescript
// 记忆提取流程
async function extractMemories(conversation: Conversation): Promise<Memory[]> {
  // 1. 触发条件检查
  if (!shouldExtractMemory(conversation)) {
    return []
  }
  
  // 2. 分析对话内容
  const analysis = await analyzeConversation(conversation)
  
  // 3. 识别候选记忆
  const candidates = identifyMemoryCandidates(analysis)
  
  // 4. 过滤 (排除可从代码推导的信息)
  const filtered = candidates.filter(c => {
    if (isDerivableFromCode(c)) return false
    if (isInGitHistory(c)) return false
    if (isEphemeral(c)) return false
    return true
  })
  
  // 5. 格式化记忆
  return filtered.map(formatAsMemory)
}

// 触发条件
function shouldExtractMemory(conv: Conversation): boolean {
  // 用户明确请求
  if (conv.containsExplicitRequest()) return true
  
  // 检测到重要信息
  if (detectImportantInformation(conv)) return true
  
  // 会话结束
  if (isSessionEnding(conv)) return true
  
  return false
}
```

### 3.3 记忆存储

```typescript
// 记忆存储
async function saveMemory(memory: Memory): Promise<void> {
  const memoryDir = getMemoryDir()
  
  // 1. 生成文件名
  const fileName = generateFileName(memory)
  // e.g., feedback_no_mock_in_tests.md
  
  // 2. 格式化文件内容
  const content = formatMemoryFile(memory)
  
  // 3. 写入文件
  await write(join(memoryDir, fileName), content)
  
  // 4. 更新索引
  await updateIndex(memoryDir, {
    fileName,
    name: memory.name,
    type: memory.type,
    description: memory.description,
  })
}

// 文件名生成
function generateFileName(memory: Memory): string {
  const sanitized = memory.name
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, '_')
    .replace(/^_|_$/g, '')
  return `${memory.type}_${sanitized}.md`
}

// 文件格式
function formatMemoryFile(memory: Memory): string {
  return `---
name: ${memory.name}
description: ${memory.description}
type: ${memory.type}
---

${memory.content}

**Why:** ${memory.why}
**How to apply:** ${memory.howToApply}
`
}
```

### 3.4 记忆检索

```typescript
// 记忆检索
async function retrieveMemories(query: string): Promise<Memory[]> {
  const memoryDir = getMemoryDir()
  
  // 1. 加载索引
  const index = await loadIndex(memoryDir)
  
  // 2. 检索相关记忆
  // 方法 1: 关键词匹配
  const keywordMatches = index.filter(entry =>
    entry.name.toLowerCase().includes(query.toLowerCase()) ||
    entry.description.toLowerCase().includes(query.toLowerCase())
  )
  
  // 方法 2: 语义匹配 (如果有嵌入模型)
  // const semanticMatches = await semanticSearch(query, index)
  
  // 3. 加载全文
  const memories = await Promise.all(
    keywordMatches.map(entry => loadMemory(memoryDir, entry.fileName))
  )
  
  return memories
}

// 启动时加载所有记忆
async function loadAllMemories(): Promise<Memory[]> {
  const memoryDir = getMemoryDir()
  const index = await loadIndex(memoryDir)
  
  return Promise.all(
    index.map(entry => loadMemory(memoryDir, entry.fileName))
  )
}
```

### 3.5 记忆注入

```typescript
// 将记忆注入上下文
function injectMemories(memories: Memory[]): string {
  const sections = {
    user: '## User Context\n',
    feedback: '## Working Guidelines\n',
    project: '## Project Context\n',
    reference: '## External Resources\n',
  }
  
  for (const memory of memories) {
    sections[memory.type] += `- ${memory.content}\n`
  }
  
  return Object.values(sections)
    .filter(s => s.trim().length > 0)
    .join('\n\n')
}

// 系统提示注入
function buildSystemPrompt(memories: Memory[]): string {
  const memoryContext = injectMemories(memories)
  
  return `你是一个超级 Agent 助手。

${memoryContext}

## 当前任务
{taskContext}

## 可用工具
{availableTools}
`
}
```

### 4.2 索引管理

```typescript
interface MemoryIndex {
  entries: Array<{
    fileName: string
    name: string
    type: MemoryType
    description: string
    path: string
  }>
}

// 读取索引
async function loadIndex(memoryDir: string): Promise<MemoryIndex> {
  const indexPath = join(memoryDir, 'MEMORY.md')
  const content = await read(indexPath, 'utf-8')
  return parseIndex(content)
}

// 更新索引
async function updateIndex(
  memoryDir: string,
  entry: MemoryIndexEntry
): Promise<void> {
  const index = await loadIndex(memoryDir)
  
  // 检查是否已存在
  const existingIndex = index.entries.findIndex(
    e => e.name === entry.name
  )
  
  if (existingIndex >= 0) {
    // 更新现有条目
    index.entries[existingIndex] = {
      ...index.entries[existingIndex],
      ...entry,
    }
  } else {
    // 添加新条目
    index.entries.push(entry)
  }
  
  // 写回索引
  await write(join(memoryDir, 'MEMORY.md'), formatIndex(index))
}

// 格式化索引
function formatIndex(index: MemoryIndex): string {
  const header = '# Memory Index\n\n'
  const entries = index.entries
    .map(e => `- [${e.name}](${e.fileName}) — ${e.description}`)
    .join('\n')
  
  return header + entries
}
```

### 4.3 索引限制

MEMORY.md 索引有以下限制（来自 Claude Code 的实现）：

```typescript
const MAX_ENTRYPOINT_LINES = 200
const MAX_ENTRYPOINT_BYTES = 25_000  // ~25KB

// 截断处理
function truncateEntrypoint(content: string): string {
  const lines = content.split('\n')
  
  if (lines.length > MAX_ENTRYPOINT_LINES) {
    content = lines.slice(0, MAX_ENTRYPOINT_LINES).join('\n')
  }
  
  if (content.length > MAX_ENTRYPOINT_BYTES) {
    // 在最后一个换行符处截断
    const cutAt = content.lastIndexOf('\n', MAX_ENTRYPOINT_BYTES)
    content = content.slice(0, cutAt > 0 ? cutAt : MAX_ENTRYPOINT_BYTES)
  }
  
  return content
}
```

## 6. 最佳实践

### 6.1 记忆写入

1. **及时写入**：发现有价值的信息立即保存
2. **具体描述**：description 字段要具体，便于检索
3. **包含原因**：Why 字段说明为什么重要
4. **可操作**：How to apply 字段说明如何应用

### 6.2 记忆组织

1. **按语义组织**：相关主题放在一起
2. **保持简洁**：每条记忆专注一个点
3. **避免重复**：写入前检查是否有相似记忆
4. **定期清理**：删除过时或不准确的记忆

### 6.3 记忆检索

1. **精确关键词**：使用具体的术语而非泛泛的词
2. **组合检索**：同时检索 name、description、content
3. **相关性排序**：按匹配度排序结果
