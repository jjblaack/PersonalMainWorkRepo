# 管家模块设计

> 管家 (Butler) 是超级 Agent 的调度中心和监督者，负责接收指令、分解任务、分配工人、监督进度、验收结果。

---

## 1. 管家的职责和定位

### 1.1 核心职责

```
┌─────────────────────────────────────────────────────────┐
│                    管家 (Butler)                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 指令接收 (Instruction Reception)                    │
│     - 监听人类消息和指令                                │
│     - 解析意图和类型                                    │
│     - 分类处理                                          │
│                                                         │
│  2. 任务分解 (Task Decomposition)                       │
│     - 将大目标拆分为可执行的子任务                      │
│     - 识别任务间的依赖关系                              │
│     - 确定执行优先级                                    │
│                                                         │
│  3. 工人分配 (Worker Assignment)                        │
│     - 从工人池获取合适的工人                            │
│     - 构建自包含的工人提示词                            │
│     - 下达执行指令                                      │
│                                                         │
│  4. 进度监督 (Progress Supervision)                     │
│     - 追踪每个任务的执行状态                            │
│     - 检测超时和异常                                    │
│     - 处理工人失败                                      │
│                                                         │
│  5. 结果验收 (Result Verification)                      │
│     - 检查任务完成质量                                  │
│     - 运行验收测试                                      │
│     - 通过/不通过决策                                   │
│                                                         │
│  6. 记忆管理 (Memory Management)                        │
│     - 从对话中提取有价值的信息                          │
│     - 写入长期记忆                                      │
│     - 更新任务历史                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 管家的定位

**管家不直接干活**，它的核心价值是：
- **理解**：理解人类意图，理解任务要求
- **分解**：将复杂目标拆分为可执行步骤
- **协调**：协调多个工人并行工作
- **监督**：确保每个任务有始有终
- **验收**：保证工作质量

**类比**：
- 管家 = 项目经理 + 技术负责人
- 工人 = 执行工程师
- 人类 = 客户/产品负责人

## 3. 指令解析器

### 3.1 指令分类

```typescript
// 指令类型
type InstructionType =
  | 'query'           // 查询类 (不需要执行任务)
  | 'task'            // 任务类 (需要执行)
  | 'configuration'   // 配置类
  | 'meta'            // 元指令 (关于系统本身)

// 指令优先级
type InstructionPriority =
  | 'urgent'          // 紧急 (立即处理)
  | 'high'            // 高
  | 'normal'          // 普通
  | 'low'             // 低 (空闲时处理)

// 指令结构
interface ParsedInstruction {
  type: InstructionType
  priority: InstructionPriority
  content: string
  metadata: {
    source: 'human' | 'scheduled' | 'event'
    timestamp: number
    userId?: string
  }
  // 任务类指令特有字段
  taskSpec?: {
    goal: string
    constraints?: string[]
    successCriteria?: string[]
    deadline?: number
  }
}
```

### 3.2 指令解析逻辑

```typescript
class InstructionParser {
  // 解析指令
  async parse(rawInstruction: string): Promise<ParsedInstruction> {
    // 1. 检查是否是元指令
    if (rawInstruction.startsWith('/')) {
      return this.parseMetaInstruction(rawInstruction)
    }
    
    // 2. 检查是否是查询类
    if (this.isQuery(rawInstruction)) {
      return {
        type: 'query',
        priority: 'normal',
        content: rawInstruction,
        metadata: { source: 'human', timestamp: Date.now() },
      }
    }
    
    // 3. 使用 LLM 分析指令类型和意图
    const analysis = await this.analyzeWithLLM(rawInstruction)
    
    return {
      type: analysis.type,
      priority: this.extractPriority(rawInstruction),
      content: rawInstruction,
      metadata: { source: 'human', timestamp: Date.now() },
      taskSpec: analysis.taskSpec,
    }
  }
  
  // 使用 LLM 分析指令
  private async analyzeWithLLM(instruction: string): Promise<AnalysisResult> {
    const prompt = `
分析以下指令的类型和意图：

指令：${instruction}

请分析：
1. 这是什么类型？(query/task/configuration)
2. 是否需要执行任务？
3. 如果是任务，目标是什么？
4. 有什么约束条件？
5. 成功的标准是什么？
6. 优先级是什么？

以 JSON 格式返回。
`
    const response = await llm.chat([{ role: 'user', content: prompt }])
    return JSON.parse(response.content)
  }
  
  // 检查是否是查询
  private isQuery(instruction: string): boolean {
    const queryPatterns = [
      /^什么/, /^怎么/, /^为什么/, /^如何/,  // 疑问词开头
      /^能.*吗/, /^可以.*吗/,                // 能否类问题
      /^请(问|解释|说明)/,                   // 请求解释
    ]
    
    return queryPatterns.some(pattern => pattern.test(instruction))
  }
  
  // 提取优先级
  private extractPriority(instruction: string): InstructionPriority {
    const lower = instruction.toLowerCase()
    
    if (/(立刻 | 马上 | 紧急 | 急 |urgent|asap)/i.test(lower)) {
      return 'urgent'
    }
    if (/(重要 | 优先|high priority)/i.test(lower)) {
      return 'high'
    }
    if (/(有空 | 方便 |low priority)/i.test(lower)) {
      return 'low'
    }
    
    return 'normal'
  }
}
```

### 4.2 任务分解器实现

```typescript
class TaskDecomposer {
  private llm: LLMProvider
  
  constructor(llm: LLMProvider) {
    this.llm = llm
  }
  
  // 分解目标
  async decompose(goal: string, context?: DecompositionContext): Promise<SubTask[]> {
    const prompt = this.buildDecompositionPrompt(goal, context)
    
    const response = await this.llm.chat([
      { role: 'system', content: DECOMPOSITION_SYSTEM_PROMPT },
      { role: 'user', content: prompt },
    ])
    
    const tasks = JSON.parse(response.content) as SubTask[]
    
    // 验证和修正任务
    return this.validateAndFixTasks(tasks)
  }
  
  // 构建分解提示词
  private buildDecompositionPrompt(
    goal: string,
    context?: DecompositionContext
  ): string {
    let prompt = `将以下目标分解为可执行的子任务：

目标：${goal}
`
    
    if (context?.constraints?.length) {
      prompt += `\n约束条件:\n${context.constraints.join('\n')}\n`
    }
    
    if (context?.successCriteria?.length) {
      prompt += `\n成功标准:\n${context.successCriteria.join('\n')}\n`
    }
    
    prompt += `
请将目标分解为 3-10 个子任务，每个任务应该是：
1. 原子的 - 可以独立执行
2. 具体的 - 有明确的完成标准
3. 可验证的 - 可以检查是否完成

返回 JSON 格式：
[
  {
    "subject": "任务主题 (命令式短语)",
    "description": "任务详细描述",
    "type": "research|implementation|test|documentation",
    "estimatedMinutes": 30,
    "dependencies": [0, 1]  // 依赖的任务索引
  }
]
`
    
    return prompt
  }
  
  // 验证和修正任务
  private validateAndFixTasks(tasks: SubTask[]): SubTask[] {
    // 1. 检查循环依赖
    if (this.hasCircularDependency(tasks)) {
      throw new Error('任务分解包含循环依赖')
    }
    
    // 2. 确保依赖索引有效
    for (const task of tasks) {
      task.dependencies = task.dependencies?.filter(
        idx => idx >= 0 && idx < tasks.length
      )
    }
    
    // 3. 添加默认值
    for (const task of tasks) {
      task.type ||= 'implementation'
      task.estimatedMinutes ||= 30
      task.dependencies ||= []
    }
    
    return tasks
  }
  
  // 检查循环依赖
  private hasCircularDependency(tasks: SubTask[]): boolean {
    const visited = new Set<number>()
    const stack = new Set<number>()
    
    const dfs = (index: number): boolean => {
      if (stack.has(index)) return true
      if (visited.has(index)) return false
      
      visited.add(index)
      stack.add(index)
      
      const task = tasks[index]
      for (const depIndex of task.dependencies || []) {
        if (dfs(depIndex)) return true
      }
      
      stack.delete(index)
      return false
    }
    
    for (let i = 0; i < tasks.length; i++) {
      if (dfs(i)) return true
    }
    
    return false
  }
}

// 分解系统提示词
const DECOMPOSITION_SYSTEM_PROMPT = `你是一个任务分解专家。你的工作是将复杂目标拆分为可执行的子任务。

分解原则：
1. 原子性：每个任务应该专注于单一目标
2. 独立性：任务之间应该尽量减少依赖
3. 可执行性：每个任务应该能够被一个工人在合理时间内完成
4. 可验证性：每个任务应该有明确的完成标准

任务类型：
- research: 研究、探索、信息收集
- implementation: 实现、编码、修改
- test: 测试、验证
- documentation: 文档、注释、说明

依赖关系：
- 使用数组索引表示依赖 (0 表示第一个任务)
- 研究类任务通常没有依赖
- 实现类任务通常依赖研究任务
- 测试类任务通常依赖实现任务`
```

## 5. 工人分配器

### 5.1 工人选择策略

```typescript
class WorkerAllocator {
  private workerPool: WorkerPool
  private taskManager: TaskManager
  
  // 分配任务给工人
  async allocate(task: TaskSpec): Promise<WorkerAssignment> {
    // 1. 检查是否有空闲工人
    const idleWorker = await this.findIdleWorker()
    
    if (idleWorker) {
      // 重用空闲工人 (有上下文优势)
      return {
        worker: idleWorker,
        mode: 'continue',
        reason: 'Reuse idle worker',
      }
    }
    
    // 2. 创建新工人
    if (await this.workerPool.canCreateWorker()) {
      const newWorker = await this.workerPool.createWorker()
      return {
        worker: newWorker,
        mode: 'spawn',
        reason: 'New worker created',
      }
    }
    
    // 3. 等待工人释放
    return {
      worker: null,
      mode: 'queue',
      reason: 'Waiting for worker availability',
      estimatedWait: this.estimateWaitTime(),
    }
  }
  
  // 查找空闲工人
  private async findIdleWorker(): Promise<Worker | null> {
    const workers = await this.workerPool.listWorkers()
    
    // 优先选择同类型的空闲工人
    for (const worker of workers) {
      if (worker.status === 'idle') {
        return worker
      }
    }
    
    return null
  }
  
  // 构建工人提示词
  buildWorkerPrompt(task: TaskSpec, context?: TaskContext): string {
    const sections: string[] = []
    
    // 1. 任务主题和描述
    sections.push(`## Task
**Subject**: ${task.subject}

**Description**: ${task.description}
`)
    
    // 2. 背景上下文
    if (context?.background) {
      sections.push(`## Context
${context.background}
`)
    }
    
    // 3. 相关文件
    if (context?.files?.length) {
      sections.push(`## Files Involved
${context.files.map(f => `- ${f.path}: ${f.summary}`).join('\n')}
`)
    }
    
    // 4. 成功标准
    if (task.successCriteria?.length) {
      sections.push(`## Success Criteria
${task.successCriteria.map(c => `- [ ] ${c}`).join('\n')}
`)
    }
    
    // 5. 输出格式
    sections.push(`## Output Format
Please provide:
1. Summary of work done
2. List of modified files (if any)
3. Test results (if applicable)
4. Commit hash (if code changes made)
5. Any issues encountered
`)
    
    // 6. 约束条件
    if (task.constraints?.length) {
      sections.push(`## Constraints
${task.constraints.map(c => `- ${c}`).join('\n')}
`)
    }
    
    return sections.join('\n')
  }
}

interface WorkerAssignment {
  worker: Worker | null
  mode: 'continue' | 'spawn' | 'queue'
  reason: string
  estimatedWait?: number
}
```

```typescript
// 提示词质量检查清单
function checkPromptQuality(prompt: string): PromptQualityReport {
  const report: PromptQualityReport = {
    score: 0,
    issues: [],
    suggestions: [],
  }
  
  // 检查 1: 是否包含任务主题
  if (!prompt.includes('Subject') && !prompt.includes('任务')) {
    report.issues.push('缺少任务主题')
    report.suggestions.push('添加明确的任务主题')
  }
  
  // 检查 2: 是否包含具体描述
  if (prompt.length < 100) {
    report.issues.push('描述可能过于简略')
    report.suggestions.push('添加更详细的任务描述')
  }
  
  // 检查 3: 是否包含成功标准
  if (!prompt.includes('Success') && !prompt.includes('完成')) {
    report.issues.push('缺少成功标准')
    report.suggestions.push('明确说明什么是"完成"')
  }
  
  // 检查 4: 是否包含上下文
  if (!prompt.includes('Context') && !prompt.includes('背景')) {
    report.issues.push('缺少上下文')
    report.suggestions.push('添加相关背景信息')
  }
  
  // 检查 5: 是否包含文件路径
  const hasFilePath = /src\/|app\/|lib\//.test(prompt)
  if (!hasFilePath && prompt.includes('实现')) {
    report.issues.push('缺少具体文件路径')
    report.suggestions.push('添加涉及的文件路径')
  }
  
  // 计算分数
  report.score = 100 - report.issues.length * 20
  
  return report
}

  score: number
  issues: string[]
  suggestions: string[]
```

## 6. 进度监督器

### 6.1 监督策略

```typescript
class ProgressMonitor {
  private pollingInterval: number = 30000  // 30 秒
  private timeoutThreshold: number = 3600000  // 60 分钟
  
  // 开始监督任务
  startSupervising(taskId: string): SupervisionHandle {
    const handle = new SupervisionHandle(taskId)
    
    // 启动轮询
    const intervalId = setInterval(async () => {
      try {
        await this.checkProgress(taskId, handle)
      } catch (error) {
        logger.error(`Progress check failed for ${taskId}:`, error)
      }
    }, this.pollingInterval)
    
    handle.onComplete(() => clearInterval(intervalId))
    
    return handle
  }
  
  // 检查进度
  private async checkProgress(
    taskId: string,
    handle: SupervisionHandle
  ): Promise<void> {
    const task = await this.taskManager.getTask(taskId)
    if (!task) return
    
    const elapsed = Date.now() - task.startedAt!
    
    // 检查超时
    if (elapsed > this.timeoutThreshold) {
      await this.handleTimeout(task)
      return
    }
    
    // 检查工人状态
    if (task.owner) {
      const workerAlive = await this.workerPool.isWorkerAlive(task.owner)
      if (!workerAlive) {
        await this.handleWorkerDeath(task)
        return
      }
    }
    
    // 报告进度
    handle.emitProgress({
      taskId,
      status: task.status,
      elapsed,
      estimatedRemaining: this.estimateRemaining(task),
    })
  }
  
  // 处理超时
  private async handleTimeout(task: Task): Promise<void> {
    logger.warn(`Task ${task.id} timed out after ${task.startedAt}`)
    
    // 停止工人
    if (task.owner) {
      await this.workerPool.killWorker(task.owner)
    }
    
    // 标记为失败
    await this.taskManager.failTask(task.id, 'Task timed out')
    
    // 通知管家
    eventBus.publish('task:timeout', { task })
  }
  
  // 处理工人死亡
  private async handleWorkerDeath(task: Task): Promise<void> {
    logger.warn(`Worker for task ${task.id} died`)
    
    // 工人已死，任务需要重新分配
    task.owner = undefined
    task.status = 'pending'
    await this.taskManager.updateTask(task)
    
    // 通知管家重新分配
    eventBus.publish('task:reassign', { task })
  }
  
  // 估算剩余时间
  private estimateRemaining(task: Task): number {
    const elapsed = Date.now() - task.startedAt!
    const estimated = task.metadata?.estimatedDuration || this.timeoutThreshold
    
    return Math.max(0, estimated - elapsed)
  }
}

class SupervisionHandle {
  private listeners: Set<(progress: ProgressUpdate) => void> = new Set()
  private completeListeners: Set<() => void> = new Set()
  
  constructor(public readonly taskId: string) {}
  
  onProgress(fn: (progress: ProgressUpdate) => void): void {
    this.listeners.add(fn)
  }
  
  onComplete(fn: () => void): void {
    this.completeListeners.add(fn)
  }
  
  emitProgress(update: ProgressUpdate): void {
    if (update.status === 'completed' || update.status === 'failed') {
      this.completeListeners.forEach(fn => fn())
    }
    this.listeners.forEach(fn => fn(update))
  }
}

interface ProgressUpdate {
  taskId: string
  status: TaskStatus
  elapsed: number
  estimatedRemaining?: number
}
```

```typescript
class ResultVerifier {
  // 验收任务结果
  async verify(task: Task): Promise<VerificationResult> {
    const checks: VerificationCheck[] = []
    
    // 1. 检查输出存在
    const outputExists = await this.checkOutputExists(task)
    checks.push({
      name: '输出存在',
      passed: outputExists,
      details: outputExists ? '输出文件存在' : '输出文件不存在',
    })
    
    // 2. 检查输出质量
    if (outputExists) {
      const qualityCheck = await this.checkOutputQuality(task)
      checks.push(qualityCheck)
    }
    
    // 3. 运行相关测试
    if (task.type === 'implementation') {
      const testResult = await this.runTests(task)
      checks.push({
        name: '测试通过',
        passed: testResult.passed,
        details: testResult.details,
      })
    }
    
    // 4. 代码规范检查
    if (task.type === 'implementation') {
      const lintResult = await this.runLint(task)
      checks.push({
        name: '代码规范',
        passed: lintResult.passed,
        details: lintResult.details,
      })
    }
    
    // 5. 类型检查
    if (task.type === 'implementation') {
      const typecheckResult = await this.runTypecheck(task)
      checks.push({
        name: '类型检查',
        passed: typecheckResult.passed,
        details: typecheckResult.details,
      })
    }
    
    // 汇总结果
    const passed = checks.every(c => c.passed)
    
    return {
      passed,
      checks,
      summary: this.generateSummary(checks),
      action: passed ? 'complete' : 'retry',
    }
  }
  
  // 检查输出存在
  private async checkOutputExists(task: Task): Promise<boolean> {
    if (!task.outputFile) return false
    
    try {
      await access(task.outputFile)
      const content = await read(task.outputFile, 'utf-8')
      return content.trim().length > 0
    } catch {
      return false
    }
  }
  
  // 检查输出质量 (使用 LLM)
  private async checkOutputQuality(task: Task): Promise<VerificationCheck> {
    const output = await read(task.outputFile, 'utf-8')
    
    const prompt = `评估以下工作输出的质量：

任务：${task.subject}
描述：${task.description}

输出：
${output}

评估标准：
1. 是否完成了任务要求？
2. 输出是否完整？
3. 是否有明显的错误或遗漏？

以 JSON 格式返回：
{
  "passed": boolean,
  "score": number,  // 0-100
  "feedback": string
}
`
    
    const response = await this.llm.chat([{ role: 'user', content: prompt }])
    const result = JSON.parse(response.content)
    
    return {
      name: '输出质量',
      passed: result.passed,
      details: result.feedback,
    }
  }
  
  // 运行测试
  private async runTests(task: Task): Promise<TestResult> {
    // 查找相关测试文件
    const testFiles = await this.findRelatedTests(task)
    
    if (testFiles.length === 0) {
      return { passed: true, details: '无相关测试' }
    }
    
    // 运行测试
    const result = await this.runTestCommand(testFiles)
    
    return {
      passed: result.exitCode === 0,
      details: result.output,
    }
  }
  
  // 运行代码规范检查
  private async runLint(task: Task): Promise<LintResult> {
    const modifiedFiles = await this.getModifiedFiles(task)
    
    if (modifiedFiles.length === 0) {
      return { passed: true, details: '无代码修改' }
    }
    
    const result = await this.runLintCommand(modifiedFiles)
    
    return {
      passed: result.exitCode === 0,
      details: result.output || '无问题',
    }
  }
  
  // 运行类型检查
  private async runTypecheck(task: Task): Promise<TypecheckResult> {
    const result = await this.runTypecheckCommand()
    
    return {
      passed: result.exitCode === 0,
      details: result.output || '类型检查通过',
    }
  }
  
  private generateSummary(checks: VerificationCheck[]): string {
    const passed = checks.filter(c => c.passed).length
    const total = checks.length
    
    if (passed === total) {
      return `所有 ${total} 项验收检查通过`
    } else {
      return `${passed}/${total} 项检查通过，${total - passed} 项失败`
    }
  }
}

interface VerificationResult {
  passed: boolean
  checks: VerificationCheck[]
  summary: string
  action: 'complete' | 'retry' | 'manual_review'
}

interface VerificationCheck {
  name: string
  passed: boolean
  details: string
}
```

```typescript
对话：

提取以下类型的记忆：
1. user - 用户背景、角色、偏好

3. project - 项目背景、决策、正在进行的工作
4. reference - 外部资源引用 (工具、库、文档等)

对于每条记忆，提供：
- name: 简短的名称
- type: 记忆类型
- content: 记忆内容
- why: 为什么值得记忆
- howToApply: 如何应用

以 JSON 数组格式返回。
`

    const candidates = JSON.parse(response.content) as MemoryCandidate[]
    
    // 过滤低质量候选
    return candidates.filter(c => this.isHighQuality(c))
  }
  
  // 判断记忆质量
  private isHighQuality(candidate: MemoryCandidate): boolean {
    // 检查是否具体
    if (candidate.content.length < 20) {
      return false
    }
    
    // 检查是否可从代码推导
    if (this.isDerivableFromCode(candidate.content)) {
      return false
    }
    
    // 检查是否是临时信息
    if (this.isEphemeral(candidate.content)) {
      return false
    }
    
    return true
  }
  
  // 检查是否可从代码推导
  private isDerivableFromCode(content: string): boolean {
    const patterns = [
      /function\s+\w+/,
      /class\s+\w+/,
      /interface\s+\w+/,
      /import.*from/,
    ]
    
    return patterns.some(p => p.test(content))
  }
  
  // 检查是否是临时信息
  private isEphemeral(content: string): boolean {
    const ephemeralPatterns = [
      /^正在/,
      /^暂时/,
      /^目前/,
      /稍后/,
      /等一下/,
    ]
    
    return ephemeralPatterns.some(p => p.test(content))
  }
  
  // 保存记忆
  async saveMemory(candidate: MemoryCandidate): Promise<void> {
    const memory: Memory = {
      name: candidate.name,
      type: candidate.type,
      content: candidate.content,
      why: candidate.why,
      howToApply: candidate.howToApply,
      createdAt: Date.now(),
    }
    
    await this.memoryManager.save(memory)
  }
```

## 9. 管家模块完整示例

### 9.1 完整工作流程

```typescript
class Butler {
  private instructionParser: InstructionParser
  private taskDecomposer: TaskDecomposer
  private workerAllocator: WorkerAllocator
  private progressMonitor: ProgressMonitor
  private resultVerifier: ResultVerifier
  private memoryExtractor: MemoryExtractor
  
  // 处理指令
  async handleInstruction(instruction: string): Promise<void> {
    // 1. 解析指令
    const parsed = await this.instructionParser.parse(instruction)
    
    // 2. 根据类型处理
    switch (parsed.type) {
      case 'query':
        await this.handleQuery(parsed.content)
        break
        
      case 'task':
        await this.handleTask(parsed.taskSpec!)
        break
        
      case 'configuration':
        await this.handleConfiguration(parsed.content)
        break
        
      case 'meta':
        await this.handleMetaInstruction(parsed.content)
        break
    }
  }
  
  // 处理任务类指令
  private async handleTask(spec: TaskSpec): Promise<void> {
    logger.info(`Handling task: ${spec.goal}`)
    
    // 1. 分解任务
    const subTasks = await this.taskDecomposer.decompose(spec.goal, {
      constraints: spec.constraints,
      successCriteria: spec.successCriteria,
    })
    logger.info(`Decomposed into ${subTasks.length} subtasks`)
    
    // 2. 创建任务
    const tasks: Task[] = []
    for (const subTask of subTasks) {
      const task = await this.taskManager.createTask({
        subject: subTask.subject,
        description: subTask.description,
        type: subTask.type,
        estimatedDuration: subTask.estimatedMinutes * 60 * 1000,
      })
      tasks.push(task)
    }
    
    // 3. 建立依赖关系
    for (let i = 0; i < tasks.length; i++) {
      const subTask = subTasks[i]
      if (subTask.dependencies?.length) {
        const blockedBy = subTask.dependencies.map(idx => tasks[idx].id)
        await this.taskManager.addDependencies(tasks[i].id, blockedBy)
      }
    }
    
    // 4. 启动可执行的任务
    const executableTasks = await this.taskManager.getExecutableTasks()
    for (const task of executableTasks) {
      await this.startTask(task)
    }
  }
  
  // 启动单个任务
  private async startTask(task: Task): Promise<void> {
    logger.info(`Starting task: ${task.id}`)
    
    // 1. 获取工人
    const assignment = await this.workerAllocator.allocate({
      subject: task.subject,
      description: task.description,
      successCriteria: task.successCriteria,
    })
    
    if (assignment.mode === 'queue') {
      logger.info(`Task ${task.id} queued, waiting for worker...`)
      return
    }
    
    // 2. 构建提示词
    const prompt = this.workerAllocator.buildWorkerPrompt(task, {
      background: await this.buildTaskContext(task),
    })
    
    // 3. 分配任务
    await assignment.worker!.assign({
      subject: task.subject,
      description: task.description,
      context: prompt,
    })
    
    // 4. 更新任务状态
    await this.taskManager.updateTaskStatus(task.id, 'in_progress', {
      owner: assignment.worker!.id,
    })
    
    // 5. 开始监督
    const handle = this.progressMonitor.startSupervising(task.id)
    
    handle.onProgress(async (update) => {
      logger.info(`Task ${task.id} progress: ${update.status}`)
    })
    
    handle.onComplete(async () => {
      // 6. 验收结果
      const completedTask = await this.taskManager.getTask(task.id)
      const verification = await this.resultVerifier.verify(completedTask!)
      
      if (verification.passed) {
        await this.taskManager.updateTaskStatus(task.id, 'completed')
        
        // 7. 提取记忆
        const memories = await this.memoryExtractor.extract(
          await this.getTaskConversation(task.id)
        )
        for (const memory of memories) {
          await this.memoryExtractor.saveMemory(memory)
        }
        
        // 8. 启动依赖此任务的其他任务
        await this.taskManager.startBlockedByTasks(task.id)
      } else {
        logger.warn(`Task ${task.id} verification failed: ${verification.summary}`)
        
        // 尝试重试
        if (verification.action === 'retry') {
          await this.retryTask(task)
        } else {
          await this.taskManager.updateTaskStatus(task.id, 'failed')
        }
      }
    })
  }
  
  // 重试任务
  private async retryTask(task: Task): Promise<void> {
    const retryCount = task.metadata?.retryCount || 0
    
    if (retryCount >= MAX_RETRIES) {
      logger.error(`Task ${task.id} exceeded max retries`)
      await this.taskManager.updateTaskStatus(task.id, 'failed')
      return
    }
    
    // 继续同一工人进行修复
    if (task.owner) {
      const worker = await this.workerPool.getWorker(task.owner)
      await worker!.continue('验收未通过，请修复问题')
    } else {
      await this.taskManager.updateTaskStatus(task.id, 'pending')
    }
  }
  
  // 构建任务上下文
  private async buildTaskContext(task: Task): Promise<string> {
    // 从记忆中检索相关信息
    const memories = await this.memoryManager.search(task.subject)
    
    if (memories.length === 0) {
      return 'No specific context available.'
    }
    
    return `
## Relevant Memories
${memories.map(m => `- [${m.type}] ${m.content}`).join('\n')}
`
  }
}
```

## 11. 最佳实践

### 11.1 任务分解最佳实践

1. **从研究开始**：先了解再行动
2. **并行优先**：识别可以并行执行的任务
3. **明确依赖**：依赖关系要清晰
4. **合理估算**：不要过于乐观

### 11.2 工人提示词最佳实践

1. **自包含**：工人看不到对话历史，提示词必须包含所有必要信息
2. **具体路径**：包含具体的文件路径和行号
3. **明确完成标准**：说明什么是"完成"
4. **添加目的说明**：让工人理解任务的目的

### 11.3 监督最佳实践

1. **主动监督**：不要被动等待结果
2. **早期预警**：在超时前预警
3. **优雅失败**：失败时保留上下文
4. **及时恢复**：可恢复的错误自动重试
