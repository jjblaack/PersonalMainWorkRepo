## 核心洞见

基于对 Claude Code 项目的深度分析，得出以下核心洞见：

### 1. 架构设计方面
- **分层清晰**：入口层→命令层→工具层→服务层→基础设施层
- **工具系统设计**：每个工具是自包含模块，定义自己的 Schema、权限、执行逻辑
- **Bridge 系统**：轮询 + 心跳模式比长连接更可靠

### 2. 记忆系统方面
- **四种记忆类型**：user、feedback、project、reference 是完备的分类
- **什么不该保存**：可从代码推导的信息、Git 历史已有的信息、临时任务状态
- **索引限制**：MEMORY.md 限制在 200 行/25KB，超出需要截断

### 3. 任务系统方面
- **状态机设计**：pending→in_progress→completed/failed/killed
- **终端状态不可逆**：completed/failed/killed 的任务不能随意修改
- **输出持久化**：每个任务有 outputFile 用于记录结果

### 4. 24 小时运行方面
- **状态外置**：所有关键状态存储在外部，崩溃后可恢复
- **轮询模型**：5 秒轮询间隔，指数退避重连
- **心跳保活**：60 秒心跳，防止会话过期
