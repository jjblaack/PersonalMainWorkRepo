## 核心概念

| 概念 | 说明 |
|------|------|
| **超级 Agent** | 对外表现为单一 Agent，对内包含管家、工人、任务的层级结构 |
| **管家** | 随时候命的调度器，负责接收指令、分配任务、监督执行 |
| **工人** | 实际执行任务的 Agent 实例，可以是 LLM 会话或工具执行器 |
| **记忆** | 分层存储系统，包含短期工作记忆和长期持久化记忆 |
| **任务闭环** | 从创建、执行、验收到归档的完整生命周期管理 |

---

## 设计原则

1. **简洁优先**：越简单的系统越可靠，适合 24 小时运行
2. **状态外置**：关键状态持久化，崩溃后可恢复
3. **监督分离**：管家不直接干活，专注于监督和调度
4. **记忆分层**：短期记忆靠上下文，长期记忆靠存储
5. **质量内建**：验收机制内置于任务流程，而非事后补救
