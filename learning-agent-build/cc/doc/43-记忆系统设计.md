## 1. 记忆系统概述

### 1.1 记忆系统的职责

记忆系统允许 Claude Code 在会话之间持久化信息，实现：

- **用户偏好记忆**: 记住用户的编码习惯、偏好设置
- **项目知识**: 项目结构、技术栈、关键决策
- **反馈指导**: 从用户反馈中学习，改进后续行为
- **上下文延续**: 跨会话保持上下文连续性

### 1.2 记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│  Claude Code 会话                                            │
│  - 用户交互                                                  │
│  - 工具执行                                                  │
│  - 代码变更                                                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  记忆提取服务 (services/extractMemories/)                   │
│  - 分析对话内容                                              │
│  - 识别有价值的信息                                          │
│  - 生成记忆记录                                              │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  MEMORY.md 索引                                              │
│  - memory/user_xxx.md                                        │
│  - memory/feedback_xxx.md                                    │
│  - memory/project_xxx.md                                     │
│  - memory/reference_xxx.md                                   │
└─────────────────────────────────────────────────────────────┘
```

## 2. 记忆类型详解

### 2.1 四种记忆类型

| 类型 | 文件前缀 | 用途 | 示例 |
|------|----------|------|------|
| **user** | user_ | 用户角色、偏好、知识 | "用户是数据科学家，偏爱 pandas" |
| **feedback** | feedback_ | 用户反馈指导 | "测试不要用 mock，要真实数据库" |
| **project** | project_ | 项目相关信息 | "正在重构认证模块，法律合规驱动" |
| **reference** | reference_ | 外部资源引用 | "Bug 追踪在 Linear 项目 INGEST" |

### 2.2 记忆文件格式

```markdown
---
name: {{记忆名称}}
description: {{一行描述}}
type: {{user|feedback|project|reference}}
---

{{记忆内容}}

**Why:** {{原因/背景}}
**How to apply:** {{如何应用}}
```

### 2.3 记忆示例

#### User 记忆示例

```markdown
---
name: data_scientist_profile
description: 用户是数据科学家，专注于数据管道
type: user
---

用户是一名数据科学家，主要工作是构建和维护数据管道。技术栈包括 Python、SQL、Apache Spark。

**Why:** 用户在对话中明确说明了自己的角色和技术栈。
**How to apply:** 在解释概念时使用数据管道相关的类比，优先推荐数据处理相关的库和工具。
```

#### Feedback 记忆示例

```markdown
---
name: no_mock_in_tests
description: 测试使用真实数据库，不要用 mock
type: feedback
---

测试应该连接真实的 PostgreSQL 数据库，不要使用 mock 或 in-memory 数据库。

**Why:** 之前 mock 测试通过了但生产环境失败，因为 mock 没有捕捉到真实的数据库行为。
**How to apply:** 编写新测试时使用 Docker 启动测试数据库，或者使用项目的 testcontainers 配置。
```

#### Project 记忆示例

```markdown
---
name: auth_refactor_compliance
description: 认证模块重构是法律合规驱动
type: project
---

正在进行的认证模块重构是为了满足新的 GDPR 合规要求，不是技术债务清理。

**Why:** 法律团队 flagged 当前 session token 存储方式不符合合规要求。
**How to apply:** 所有认证相关决策应优先考虑合规性而非开发便利性。
```

#### Reference 记忆示例

```markdown
---
name: linear_pipeline_bugs
description: 数据管道 bug 追踪在 Linear 项目 INGEST
type: reference
---

所有数据管道相关的 bug 都追踪在 Linear 项目的 "INGEST" 项目中。

**Why:** 团队选择将管道 bug 与主产品 bug 分开追踪。
**How to apply:** 查询管道 bug 或创建相关 ticket 时，使用 Linear 项目 INGEST。
```

```typescript
// src/memdir/paths.js (推断)

import { join } from 'path';

const MEMORY_DIR_NAME = 'memory';
const INDEX_FILE = 'MEMORY.md';

export function getMemoryDir(baseDir: string): string {
  return join(baseDir, MEMORY_DIR_NAME);
}

export function getIndexFilePath(baseDir: string): string {
  return join(getMemoryDir(baseDir), INDEX_FILE);
}

export function getMemoryFilePath(baseDir: string, fileName: string): string {
  return join(getMemoryDir(baseDir), fileName);
}

export function ensureMemoryDir(baseDir: string): string {
  const dir = getMemoryDir(baseDir);
  if (!existsSync(dir)) {
    mkdirSync(dir, { recursive: true });
  }
  return dir;
}
```

### 3.3 记忆目录管理

```typescript
// src/memdir/memdir.js (推断)

import { readFileSync, writeFileSync, readdirSync } from 'fs';

export function loadMemoryPrompt(memoryDir: string): string {
  const index = readIndex(memoryDir);
  const memories = loadMemories(index);
  
  return formatMemoryPrompt(memories);
}

function readIndex(memoryDir: string): MemoryIndex {
  const indexPath = getIndexFilePath(memoryDir);
  const content = readFileSync(indexPath, 'utf-8');
  return parseIndex(content);
}

function loadMemories(index: MemoryIndex): Memory[] {
  return index.entries.map(entry => {
    const content = readFileSync(entry.path, 'utf-8');
    return {
      ...entry,
      content: parseMemoryContent(content),
    };
  });
}

function formatMemoryPrompt(memories: Memory[]): string {
  const sections = {
    user: [],
    feedback: [],
    project: [],
    reference: [],
  };
  
  for (const memory of memories) {
    sections[memory.type].push(formatMemory(memory));
  }
  
  return Object.entries(sections)
    .filter(([, items]) => items.length > 0)
    .map(([type, items]) => `## ${type}\n\n${items.join('\n')}`)
    .join('\n\n');
}
```

```typescript
// 记忆提取流程 (推断)

async function extractMemories(conversation: Conversation): Promise<Memory[]> {
  // 1. 分析对话内容
  const analysis = await analyzeConversation(conversation);
  
  // 2. 识别潜在记忆
  const candidates = identifyMemoryCandidates(analysis);
  
  // 3. 过滤和格式化
  const memories = candidates
    .filter(isValidMemory)
    .map(formatAsMemory);
  
  // 4. 用户确认 (可选)
  if (config.requireConfirmation) {
    const approved = await confirmMemories(memories);
    return approved;
  }
  
  return memories;
}

async function analyzeConversation(conv: Conversation) {
  // 使用 AI 分析对话
  const result = await query([{
    role: 'user',
    content: `Analyze this conversation and identify:
1. User preferences or role information
2. Feedback about how to work
3. Project context or decisions
4. External resource references

Conversation:
${conv.messages.map(m => `${m.role}: ${m.content}`).join('\n')}`,
  }]);
  
  return parseAnalysis(result);
}
```

```typescript
// 什么应该保存 (推断)

function shouldSave(memory: PotentialMemory): boolean {
  // 应该保存
  if (memory.type === 'user' && isNonObvious) {
    return true;  // 非明显的用户信息
  }
  
  if (memory.type === 'feedback') {
    return true;  // 用户反馈通常有价值
  }
  
  if (memory.type === 'project' && hasLongTermValue) {
    return true;  // 长期项目信息
  }
  
  if (memory.type === 'reference') {
    return true;  // 外部资源引用
  }
  
  // 不应该保存
  if (memory.type === 'ephemeral') {
    return false;  // 临时任务状态
  }
  
  if (memory.derivableFromCode) {
    return false;  // 可从代码推导的信息
  }
  
  if (memory.inGitHistory) {
    return false;  // Git 历史已有
  }
  
  return false;
}
```

## 5. 记忆写入

### 5.1 写入 API

```typescript
// 记忆写入接口 (推断)

async function saveMemory(memory: Memory): Promise<void> {
  const memoryDir = ensureMemoryDir(getProjectRoot());
  
  // 1. 写入记忆文件
  const fileName = generateFileName(memory);
  const filePath = join(memoryDir, fileName);
  
  const content = formatMemoryFile(memory);
  writeFileSync(filePath, content);
  
  // 2. 更新索引
  updateIndex(memoryDir, { fileName, ...memory });
}

function formatMemoryFile(memory: Memory): string {
  return `---
name: ${memory.name}
description: ${memory.description}
type: ${memory.type}
---

${memory.content}
`;
}

function generateFileName(memory: Memory): string {
  const sanitized = memory.name
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, '_');
  return `${memory.type}_${sanitized}.md`;
}
```

### 5.2 索引更新

```typescript
// MEMORY.md 索引管理 (推断)

interface MemoryIndex {
  entries: Array<{
    fileName: string;
    name: string;
    type: MemoryType;
    path: string;
  }>;
}

function updateIndex(memoryDir: string, memory: Memory): void {
  const indexPath = getIndexFilePath(memoryDir);
  let index = readIndex(indexPath);
  
  // 添加或更新条目
  const existingIndex = index.entries.findIndex(e => e.name === memory.name);
  
  if (existingIndex >= 0) {
    index.entries[existingIndex] = {
      ...index.entries[existingIndex],
      fileName: generateFileName(memory),
    };
  } else {
    index.entries.push({
      fileName: generateFileName(memory),
      name: memory.name,
      type: memory.type,
      path: join(memoryDir, generateFileName(memory)),
    });
  }
  
  // 写回索引
  writeFileSync(indexPath, formatIndex(index));
}

function formatIndex(index: MemoryIndex): string {
  const header = `# Memory Index\n\n`;
  const entries = index.entries
    .map(e => `- [${e.name}](${e.fileName}) — ${e.type}`)
    .join('\n');
  
  return header + entries;
}
```

## 6. 记忆的生命周期

### 6.1 记忆生命周期

```
创建 ──► 使用 ──► 更新 ──► (可能) 过期/删除
  │        │       │
  │        │       └─► 发现记忆不准确时更新
  │        │
  │        └─► 会话启动时加载
  │
  └─► 用户明确请求或自动提取
```

### 6.2 记忆加载

```typescript
// 会话启动时加载记忆 (推断)

async function loadMemoriesForSession(): Promise<Memory[]> {
  const memoryDir = getMemoryDir(getProjectRoot());
  
  if (!existsSync(memoryDir)) {
    return [];
  }
  
  // 读取索引
  const index = readIndex(memoryDir);
  
  // 加载所有记忆
  const memories = index.entries.map(entry => ({
    ...entry,
    content: readFileSync(entry.path, 'utf-8'),
  }));
  
  return memories;
}
```

### 6.3 记忆注入

```typescript
// 将记忆注入系统提示 (推断)

function injectMemoriesToSystemPrompt(memories: Memory[]): string {
  const sections = {
    user: '## User Context\n',
    feedback: '## Working Guidelines\n',
    project: '## Project Context\n',
    reference: '## External Resources\n',
  };
  
  for (const memory of memories) {
    sections[memory.type] += `- ${memory.content}\n`;
  }
  
  return Object.values(sections).join('\n');
}
```

## 7. 最佳实践

### 7.1 什么应该保存

**应该保存**:
- 用户角色和背景信息
- 用户明确的工作偏好
- 从错误中学习的经验
- 非显而易见的架构决策
- 外部资源位置 (bug 追踪、仪表盘等)
- 团队约定的工作流

**不应该保存**:
- 代码模式和技术细节 (可从代码推导)
- Git 历史已有信息
- 临时任务状态
- 显而易见的偏好

### 7.2 记忆更新

当发现记忆不准确时，应该更新而不是删除：

```markdown
// 更新记忆时添加说明

---
name: xxx
description: xxx
type: xxx
updated: 2026-04-02  # 添加更新时间
---

更新后的内容...

**Why:** 更新了原因说明
**How to apply:** 更新后的应用方式
```

### 7.3 记忆组织

- **按语义组织**: 相关主题放在一起
- **保持简洁**: 每条记忆专注一个点
- **包含原因**: 说明为什么重要
- **可操作**: 说明如何应用

## 总结

记忆系统是 Claude Code 实现个性化的核心：

1. **四种类型**: user、feedback、project、reference
2. **自动提取**: AI 分析对话识别记忆
3. **持久化存储**: Markdown 文件格式
4. **索引管理**: MEMORY.md 统一索引
5. **会话注入**: 每次会话自动加载

良好的记忆管理使得 Claude Code 能够随着时间推移变得更了解用户和项目，提供更精准的帮助。
