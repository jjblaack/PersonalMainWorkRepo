## 1. 多代理系统架构

### 1.1 架构层次

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          多代理协作架构                                      │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  主代理 (Main Agent / Coordinator)                                          │
│  - 接收用户请求                                                              │
│  - 任务分析与分解                                                            │
│  - 分配任务给子代理                                                          │
│  - 聚合子代理结果                                                            │
│  - 呈现最终结果给用户                                                        │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
              ┌───────────────────────┼───────────────────────┐
              │                       │                       │
              ▼                       ▼                       ▼
┌─────────────────────────┐ ┌─────────────────────────┐ ┌─────────────────────────┐
│  子代理 A (Worker A)     │ │  子代理 B (Worker B)     │ │  子代理 C (Worker C)     │
│  - 独立会话             │ │  - 独立会话             │ │  - 独立会话             │
│  - 独立工具调用         │ │  - 独立工具调用         │ │  - 独立工具调用         │
│  - 独立 LLM 交互          │ │  - 独立 LLM 交互          │ │  - 独立 LLM 交互          │
│  - 向主代理报告         │ │  - 向主代理报告         │ │  - 向主代理报告         │
└─────────────────────────┘ └─────────────────────────┘ └─────────────────────────┘
```

### 1.2 代理类型

| 代理类型 | 描述 | 使用场景 |
|----------|------|----------|
| **Main Agent** | 主代理，与用户直接交互 | 接收用户请求，协调整体流程 |
| **Worker Agent** | 工作代理，执行具体任务 | 执行子任务，如前端开发、后端开发 |
| **Coordinator** | 协调器代理，管理多个 Worker | 复杂项目的多代理协调 |
| **Specialist Agent** | 专业代理，有特定技能 | 代码审查、测试、文档等专门任务 |

### 2.2 AgentTool 执行流程

**位置**: `src/tools/AgentTool/AgentTool.ts`

```typescript
class AgentTool extends Tool {
  name = 'Agent'
  description = 'Spawn a sub-agent to work on a task'
  
  async execute(input: AgentToolInput, context: ToolContext): Promise<ToolResult> {
    // === 阶段 1: 创建子代理会话 ===
    
    // 1.1 生成子代理会话 ID
    const sessionId = generateSessionId()
    
    // 1.2 创建子代理会话
    const childSession = await createSession({
      id: sessionId,
      parentSessionId: context.sessionId,
      initialPrompt: input.prompt,
      agentType: input.agent_type,
      model: input.model,
      permissionMode: input.permission_mode,
    })
    
    // 1.3 向父代理报告 spawned 状态
    context.onProgress?.({
      type: 'agent',
      sessionId,
      status: 'spawning',
      message: `Spawning agent for: ${input.prompt.slice(0, 50)}...`,
    })
    
    // === 阶段 2: 启动子代理 ===
    
    // 2.1 创建子代理的 QueryEngine
    const engine = new QueryEngine({
      sessionId: childSession.id,
      model: childSession.model,
      tools: context.tools,
      commands: context.commands,
      canUseTool: context.canUseTool,
      onProgress: (progress) => {
        // 转发进度到父代理
        context.onProgress?.({
          type: 'agent',
          sessionId,
          status: progress.status,
          message: progress.message,
          percentComplete: progress.percent,
        })
      },
    })
    
    // 2.2 发送初始消息
    const messages = [
      createSystemMessage(buildAgentSystemPrompt(childSession)),
      createUserMessage(input.prompt),
    ]
    
    // === 阶段 3: 执行子代理循环 ===
    
    let response: APIResponse
    let turnCount = 0
    const maxTurns = 50
    
    while (turnCount < maxTurns) {
      response = await engine.send(messages)
      
      // 检查是否完成
      if (response.stop_reason === 'end_turn') {
        break
      }
      
      // 处理 Tool Call
      if (response.stop_reason === 'tool_use') {
        const toolResults = await engine.executeToolCalls(
          response.content,
          context.canUseTool,
          context.tools
        )
        messages.push(...toolResults)
      }
      
      turnCount++
    }
    
    // === 阶段 4: 聚合并返回结果 ===
    
    // 4.1 提取子代理的最终响应
    const fullOutput = extractAssistantContent(response)
    
    // 4.2 生成摘要 (可选，使用另一个 LLM 调用)
    const summary = await generateSummary(fullOutput)
    
    // 4.3 返回结果
    return {
      success: true,
      content: [
        {
          type: 'text',
          text: summary,
        },
      ],
      data: {
        sessionId,
        fullOutput,
        tokenUsage: response.usage,
        turnCount,
      },
    }
  }
}
```

```typescript
const childSessions: Map<string, ChildAgentSession> = new Map()

function createSession(options: SessionOptions): ChildAgentSession {
  const session: ChildAgentSession = {
    id: options.id,
    parentSessionId: options.parentSessionId,
    agentType: options.agentType,
    model: options.model || 'claude-sonnet-4-5',
    status: 'spawning',
```

```typescript
async function decomposeTask(taskDescription: string): Promise<SubTask[]> {
  // 调用 AI 进行任务分解
  const response = await query([{
    role: 'system',
    content: `You are a task decomposition expert. Break down complex tasks into independent subtasks that can be executed in parallel.`,
  }, {
    role: 'user',
    content: `Decompose the following task into subtasks:\n\n${taskDescription}`,
  }])
  
  // 解析返回的子任务列表
  return parseSubtasks(response.content)
}

function parseSubtasks(content: string): SubTask[] {
  // 解析 JSON 或 Markdown 格式的子任务列表
  const subtasks: SubTask[] = []
  
  // 示例解析逻辑
  const lines = content.split('\n')
  for (const line of lines) {
    if (line.match(/^\d+\./)) {
      subtasks.push({
        id: generateSubtaskId(),
        description: line.replace(/^\d+\.\s*/, ''),
        status: 'pending',
        assignedTo: null,
      })
    }
  }
  
  return subtasks
}
```

### 3.3 代理分配

```typescript
// src/coordinator/agentAllocation.js (推断)

async function allocateAgents(subtasks: SubTask[]): Promise<Map<string, string>> {
  const allocation = new Map<string, string>()  // subtaskId -> agentId
  
  // 调用 AI 进行代理分配
  const response = await query([{
    role: 'system',
    content: `You are an agent allocation expert. Assign each subtask to the most appropriate agent based on the task requirements.`,
  }, {
    role: 'user',
    content: `Allocate agents for these subtasks:\n\n${JSON.stringify(subtasks, null, 2)}`,
  }])
  
  // 解析分配结果
  const parsed = parseAllocation(response.content)
  
  for (const [subtaskId, agentType] of parsed) {
    // 创建或获取对应类型的代理
    const agent = await getOrCreateAgent(agentType)
    allocation.set(subtaskId, agent.id)
  }
  
  return allocation
}

async function getOrCreateAgent(agentType: string): Promise<Agent> {
  // 查找已有的同类型代理
  const existing = agents.find(a => a.type === agentType && a.isAvailable)
  
  if (existing) {
    return existing
  }
  
  // 创建新代理
  return createAgent({ type: agentType })
}
```

```typescript
class TeamCreateTool extends Tool {

      success: true,
      content: [{
        type: 'text',

      }],

    }
  }
}
```

```typescript
class SendMessageTool extends Tool {
  name = 'SendMessage'
  description = 'Send a message to another agent'
  
  async execute(input: {
    recipientId: string       // 接收者 ID
    message: string           // 消息内容
    priority?: 'low' | 'normal' | 'high'
    type?: 'request' | 'response' | 'notification'
  }, context: ToolContext): Promise<ToolResult> {
    // 查找接收者
    const recipient = findAgent(input.recipientId)
    
    if (!recipient) {
      return {
        success: false,
        content: [{
          type: 'text',
          text: `Agent ${input.recipientId} not found`,
        }],
      }
    }
    
    // 构建消息
    const agentMessage: AgentMessage = {
      id: generateMessageId(),
      from: context.agentId,
      to: input.recipientId,
      content: input.message,
      type: input.type || 'notification',
      priority: input.priority || 'normal',
      timestamp: Date.now(),
    }
    
    // 发送消息
    await deliverMessage(agentMessage)
    
    return {
      success: true,
      content: [{
        type: 'text',
        text: `Message sent to agent ${input.recipientId}`,
      }],
      data: { messageId: agentMessage.id },
    }
  }
}
```

### 5.2 消息队列

```typescript
// 代理消息队列 (推断)

class AgentMessageQueue {
  private queues: Map<string, AgentMessage[]> = new Map()
  
  // 入队
  async enqueue(message: AgentMessage): Promise<void> {
    const queue = this.queues.get(message.to) || []
    queue.push(message)
    
    // 按优先级排序
    queue.sort((a, b) => {
      const priorityOrder = { high: 0, normal: 1, low: 2 }
      return priorityOrder[a.priority] - priorityOrder[b.priority]
    })
    
    this.queues.set(message.to, queue)
    
    // 通知接收者
    this.notifyRecipient(message.to)
  }
  
  // 出队
  async dequeue(agentId: string, limit = 10): Promise<AgentMessage[]> {
    const queue = this.queues.get(agentId) || []
    const messages = queue.slice(0, limit)
    
    // 从队列移除已读取的消息
    this.queues.set(agentId, queue.slice(limit))
    
    return messages
  }
  
  // 获取队列长度
  getQueueLength(agentId: string): number {
    return (this.queues.get(agentId) || []).length
  }
  
  // 通知接收者有新消息
  private notifyRecipient(agentId: string): void {
    const agent = findAgent(agentId)
    if (agent) {
      agent.notifyNewMessage()
    }
  }
}
```

```typescript
async function aggregateResults(agentResults: Map<string, AgentResult>): Promise<FinalResult> {
  // 收集所有结果
  const results = Array.from(agentResults.values())
  
  // 调用 AI 进行结果聚合

  }])

    finalReport: response.content,
    agentResults: results,
    summary: generateExecutiveSummary(results),
  }
}
```

```
用户请求："帮我重构这个 monorepo 项目，拆分成独立的服务"

这是一个超大型任务，使用 Coordinator 模式：

Coordinator  spawned 的团队：
┌─────────────────────────────────────────────────────────────┐
│  Coordinator (协调器)                                        │
│  ├── 分析项目结构                                           │
│  ├── 制定拆分策略                                           │
│  └── 协调各团队工作                                         │
├─────────────────────────────────────────────────────────────┤
│  Team A: 服务拆分团队                                        │
│  ├── Agent A1: 分析服务边界                                 │
│  ├── Agent A2: 提取共享库                                   │
│  └── Agent A3: 创建服务模板                                 │
├─────────────────────────────────────────────────────────────┤
│  Team B: 代码迁移团队                                        │
│  ├── Agent B1: 迁移用户服务                                 │
│  ├── Agent B2: 迁移订单服务                                 │
│  └── Agent B3: 迁移支付服务                                 │
├─────────────────────────────────────────────────────────────┤
│  Team C: 基础设施团队                                        │
│  ├── Agent C1: 创建 Docker 配置                               │
│  ├── Agent C2: 创建 Kubernetes manifests                    │
│  └── Agent C3: 设置 CI/CD 流水线                             │
└─────────────────────────────────────────────────────────────┘

执行流程：
1. Coordinator 分析项目，制定拆分策略
2. 各团队并行工作
3. 团队内部代理间通过消息协调
4. Coordinator 定期检查进度，解决冲突
5. 所有团队完成后，汇总结果

通信模式：
Coordinator ←→ Team Lead A ←→ Agent A1, A2, A3
Coordinator ←→ Team Lead B ←→ Agent B1, B2, B3
Coordinator ←→ Team Lead C ←→ Agent C1, C2, C3

Agent A1 ←→ Agent A2 (共享库分析结果)
Agent B1 ←→ Agent B2 (服务间接口协调)
Agent C1 ←→ Agent C2 (基础设施配置同步)
```

## 总结

多代理协作流程是 Claude Code 处理复杂任务的核心能力：

1. **任务分解**: AI 驱动的智能任务分解
2. **并行执行**: 多个代理同时工作，提高效率
3. **专业分工**: 不同代理专注不同领域
4. **代理通信**: 支持代理间消息传递和协调
5. **结果聚合**: 自动汇总所有代理的结果

这个系统使得 Claude Code 能够处理远超单个代理能力范围的复杂项目。
