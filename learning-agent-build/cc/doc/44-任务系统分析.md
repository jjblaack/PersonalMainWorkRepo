## 1. 任务系统概述

### 1.1 任务系统的职责

任务系统允许 Claude Code 追踪和管理多步骤工作：

- **进度追踪**: 记录每个步骤的状态
- **依赖管理**: 处理任务间的依赖关系
- **并行执行**: 支持多个任务并行
- **结果聚合**: 汇总所有任务的结果

### 1.2 使用场景

```
用户请求："帮我完成这个功能开发"

任务分解:
┌─────────────────────────────────────────────────────────────┐
│ Task 1: 数据库迁移                                           │
│ status: completed                                           │
├─────────────────────────────────────────────────────────────┤
│ Task 2: API 开发                                             │
│ status: in_progress                                         │
│ blockedBy: [Task 1]                                         │
├─────────────────────────────────────────────────────────────┤
│ Task 3: 前端组件                                             │
│ status: pending                                             │
│ blockedBy: [Task 2]                                         │
├─────────────────────────────────────────────────────────────┤
│ Task 4: 单元测试                                             │
│ status: pending                                             │
│ blockedBy: [Task 2, Task 3]                                 │
└─────────────────────────────────────────────────────────────┘
```

## 4. 任务状态机

### 4.1 状态定义

```typescript
type TaskStatus = 
  | 'pending'        // 初始状态，等待执行
  | 'in_progress'    // 正在执行
  | 'completed'      // 执行完成
  | 'deleted';       // 已删除
```

### 4.2 状态转换规则

```typescript
// 状态转换矩阵 (推断)

const VALID_TRANSITIONS: Record<TaskStatus, TaskStatus[]> = {
  'pending': ['in_progress', 'deleted'],
  'in_progress': ['completed', 'pending', 'deleted'],
  'completed': ['pending', 'deleted'],  // 可重新打开
  'deleted': [],  // 终端状态
};

function isValidTransition(from: TaskStatus, to: TaskStatus): boolean {
  return VALID_TRANSITIONS[from]?.includes(to) ?? false;
}

function updateTaskStatus(task: Task, newStatus: TaskStatus): Task {
  if (!isValidTransition(task.status, newStatus)) {
    throw new Error(
      `Invalid transition from ${task.status} to ${newStatus}`
    );
  }
  
  return {
    ...task,
    status: newStatus,
    updatedAt: Date.now(),
  };
}
```

### 4.3 状态检查

```typescript
// 检查任务是否可以开始 (推断)

async function canStartTask(taskId: string): Promise<boolean> {
  const task = await getTask(taskId);
  
  if (!task) return false;
  if (task.status !== 'pending') return false;
  
  // 检查所有依赖是否完成
  for (const blockedById of task.blockedBy) {
    const dependency = await getTask(blockedById);
    if (!dependency || dependency.status !== 'completed') {
      return false;
    }
  }
  
  return true;
}
```

### 5.3 循环依赖检测

```typescript
// 检测循环依赖 (推断)

function hasCircularDependency(taskId: string, tasks: Task[]): boolean {
  const visited = new Set<string>();
  const recursionStack = new Set<string>();
  
  function dfs(currentId: string): boolean {
    if (recursionStack.has(currentId)) {
      return true;  // 检测到循环
    }
    
    if (visited.has(currentId)) {
      return false;
    }
    
    visited.add(currentId);
    recursionStack.add(currentId);
    
    const task = tasks.find(t => t.id === currentId);
    if (!task) return false;
    
    for (const depId of task.blockedBy) {
      if (dfs(depId)) {
        return true;
      }
    }
    
    recursionStack.delete(currentId);
    return false;
  }
  
  return dfs(taskId);
}
```

## 6. 任务与子代理

### 6.1 任务分配给代理

```typescript
// 任务分配 (推断)

async function assignTaskToAgent(taskId: string, agentId: string): Promise<void> {
  const task = await getTask(taskId);
  
  if (!task) {
    throw new Error(`Task ${taskId} not found`);
  }
  
  // 更新任务所有者
  await updateTask({
    ...task,
    owner: agentId,
    status: 'in_progress',
  });
  
  // 通知代理
  const agent = findAgent(agentId);
  if (agent) {
    await agent.receiveTask(task);
  }
}
```

async function recordTaskOutput(taskId: string, output: string): Promise<void> {
  const outputPath = join(getTaskOutputDir(), `${taskId}.md`);
  
  // 写入输出文件
  writeFileSync(outputPath, output);

### 6.3 后台任务执行

```typescript
// 后台任务执行 (推断)

async function runTaskInBackground(taskId: string): Promise<void> {
  // 更新状态为进行中
  await updateTaskStatus(taskId, 'in_progress');
  
  // 启动后台进程
  const process = spawnTaskProcess(taskId);
  
  // 监听完成
  process.on('complete', async (result) => {
    await recordTaskOutput(taskId, result.output);
    await updateTaskStatus(taskId, 'completed');
  });
  
  process.on('error', async (error) => {
    await recordTaskError(taskId, error);
    await updateTaskStatus(taskId, 'pending');  // 可重试
  });
}
```

## 8. tasks.ts 分析

### 8.1 任务管理函数

**位置**: `src/tasks.ts`

```typescript
// src/tasks.ts (简化推断)

import { Task, TaskStatus } from './Task.js';

const tasks: Map<string, Task> = new Map();

export async function createTask(subject: string, options?: Partial<Task>): Promise<Task> {
  const task: Task = {
    id: generateTaskId(),
    subject,
    description: options?.description || '',
    activeForm: options?.activeForm,
    status: 'pending',
    blockedBy: [],
    blocks: [],
    createdAt: Date.now(),
    updatedAt: Date.now(),
  };
  
  tasks.set(task.id, task);
  return task;
}

export async function updateTask(updates: Partial<Task> & { id: string }): Promise<Task> {
  const existing = tasks.get(updates.id);
  if (!existing) {
    throw new Error(`Task ${updates.id} not found`);
  }
  
  const updated = {
    ...existing,
    ...updates,
    updatedAt: Date.now(),
  };
  
  tasks.set(updated.id, updated);
  return updated;
}

export async function listTasks(filter?: { status?: TaskStatus }): Promise<Task[]> {
  let result = Array.from(tasks.values());
  
  if (filter?.status) {
    result = result.filter(t => t.status === filter.status);
  }
  
  return result;
}

export async function getTask(id: string): Promise<Task | undefined> {
  return tasks.get(id);
}
```

## 总结

任务系统是 Claude Code 管理工作流的核心：

1. **状态管理**: pending/in_progress/completed/deleted
2. **依赖追踪**: blocks/blockedBy 关系
3. **并行执行**: 多个任务/代理同时工作
4. **输出管理**: 任务结果持久化
5. **灵活查询**: 列表/详情/输出查询

这个系统使得 Claude Code 能够清晰地追踪复杂项目的进度，并在 UI 中向用户展示实时状态。
