# code-pattern-extractor 技能

## 用途

从源代码中提取可复用的模式，包括：
1. 设计模式实现
2. 代码模板
3. 配置示例
4. 接口定义模式
5. 错误处理模式

## 使用方法

```
/code-pattern-extractor <文件路径或目录或主题>
```

## 提取的模式类型

### 1. 架构模式
- 分层架构实现
- 模块边界定义
- 依赖注入方式
- 状态管理模式

### 2. 代码模式
- 工具类实现模式
- 异步处理模式
- 流式处理模式
- 错误处理模式
- 日志记录模式

### 3. 接口模式
- Tool 接口实现
- 服务接口定义
- 事件处理器模式
- 回调/观察者模式

### 4. 配置模式
- Schema 定义
- 配置加载
- 迁移脚本
- 特性标志使用

## 输出格式

对每个提取的模式：

### 模式名称
**用途**：何时使用此模式
**实现位置**：源代码中的位置
**核心代码**：关键实现片段
**使用示例**：如何使用此模式
**注意事项**：使用时的注意点

## 执行步骤

1. 扫描目标代码
2. 识别重复出现的结构
3. 分类为已知模式类型
4. 提取核心实现
5. 编写使用说明
6. 生成模式目录
