# deep-file-analysis 技能

## 用途

对单个 TypeScript/JavaScript 文件进行深度分析，提取：
1. 文件概览（行数、导出、依赖）
2. 核心函数/类的职责和签名
3. 设计模式识别
4. 关键数据结构和接口
5. 与其他模块的依赖关系

## 使用方法

```
/deep-file-analysis <文件路径>
```

## 输出格式

### 1. 文件概览
```
文件：src/QueryEngine.ts
大小：46K 行
主要导出：QueryEngine 类、QueryConfig 接口
主要依赖：Anthropic API、工具执行器、流式处理器
```

### 2. 核心组件分析
对每个导出的类/函数：
- 职责描述
- 输入/输出签名
- 关键方法和内部逻辑

### 3. 设计模式识别
识别使用的模式，例如：
- 单例模式
- 工厂模式
- 观察者模式
- 策略模式
- 管道 - 过滤器模式

### 4. 可复用片段
提取可以复用的代码模式或配置模板

## 执行步骤

1. 读取目标文件
2. 解析 import/导出
3. 识别主要类和函数
4. 分析每个组件的职责
5. 识别设计模式
6. 提取可复用片段
7. 生成结构化报告
