## 二、模块划分

### 核心模块清单（6 个）

| 模块 | 职责 | 关键类/函数 |
|------|------|-------------|
| `cli` | CLI 入口、命令解析、参数处理 | `app`, `extract`, `batch`, `init` |
| `config` | 配置管理、验证、持久化 | `Settings`, `load_config`, `save_config` |
| `extractors` | 各类文档提取，统一接口 | `BaseExtractor`, `WebExtractor`, `PDFExtractor`, `DocxExtractor` |
| `analyzers` | 内容分析、分类、关键词 | `analyze_topic`, `extract_keywords`, `determine_hierarchy` |
| `generators` | Markdown 格式化、Wiki 生成 | `generate_markdown`, `generate_wiki_index`, `generate_toc` |
| `utils` | 日志、进度条、文件工具、HTTP | `logger`, `http_client`, `file_utils` |

### 模块依赖关系

```
cli ─┬─→ config
     ├─→ extractors ─→ utils
     ├─→ analyzers ─→ utils
     └─→ generators ─┬─→ analyzers
                     └─→ utils
```

## 四、MVP 定义

### 最小可行功能集合

```
✓ CLI 基础命令
✓ Web 提取（网页 → Markdown）
✓ PDF 提取（简单 PDF → Markdown）
✓ 输出 .md 文件到本地
✓ 基础配置支持（输出目录等）
```

### MVP 演示场景

```bash
# 场景 1: 提取网页文章
$ anything-kb https://zh.wikipedia.org/wiki/Python
✓ 提取成功，保存到 ./knowledge/wiki/Python.md

# 场景 2: 提取 PDF 文档
$ anything-kb lecture_notes.pdf
✓ PDF 提取成功，保存到 ./knowledge/docs/lecture_notes.md

# 场景 3: 查看生成的文件
$ cat ./knowledge/wiki/Python.md
# Python 编程语言
...（Markdown 内容）
```

### MVP 不做的功能

- 自动分类（MVP 阶段手动指定或平铺）
- Wiki 多级索引（MVP 阶段只有输出文件）
- DOCX/Excel 支持（Phase 2 再做）
- Claude Skill（Phase 4 再做）

## 五、技术风险点

### 风险清单

| 风险 | 可能性 | 影响 | 规避方案 |
|------|--------|------|----------|
| PDF 格式复杂（公式/表格/双栏） | 高 | 高 | 使用 marker，提供 --simple 模式，文档说明限制 |
| 网页反爬（需要 JS 渲染） | 中 | 中 | 支持自定义 headers/cookies，可选 Playwright |
| 中文编码问题 | 中 | 高 | 统一 UTF-8，充分测试各种来源 |
| 依赖安装复杂（OCR 等） | 高 | 中 | 核心依赖用纯 Python 库，OCR 作可选依赖 |
| 分类准确率低 | 高 | 中 | 初期用规则 + 用户手动调整，后期接入 AI |
| 大文件内存溢出 | 低 | 高 | 流式处理，分块读取 |
| PyPI 发布问题 | 低 | 中 | 提前测试，用成熟打包方案 |

### 风险应对策略

1. **PDF 复杂格式**: 
   - 优先使用 marker（专门做 PDF→MD）
   - 提供 --simple 模式跳过复杂处理
   - 在文档中明确说明限制

2. **依赖管理**:
   - 核心依赖：typer, requests, beautifulsoup4, markdown
   - 可选依赖：marker-pdf, mammoth, pandas (用 extras_require)

3. **分类准确率**:
   - Phase 3 先用规则（关键词匹配）
   - Phase 5 再接入 AI（spaCy/LLM）
   - 允许用户手动调整分类

## 九、决策记录

### 已确认的决策

| 决策 | 选项 | 原因 |
|------|------|------|
| CLI 框架 | typer | 基于 click，类型友好，文档好 |
| PDF 提取 | marker + pdfplumber | marker 专做 PDF→MD，pdfplumber 处理表格 |
| DOCX 提取 | mammoth | 跨平台，API 简单 |
| 配置格式 | YAML + 环境变量 | 人类可读，支持环境变量覆盖 |
| 日志库 | loguru | 简单好用，彩色输出 |
| 测试框架 | pytest | 事实标准，生态好 |
