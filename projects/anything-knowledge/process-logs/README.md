# Process Logs - 项目开发过程记录

## 用途

本目录用于存放 anything-knowledge 项目开发和交流过程中的所有记录，包括：
- 需求讨论和决策过程
- 技术方案设计文档
- 市场调研和竞品分析
- 阶段性总结
- 用户思路和想法记录

## 文档结构

```
process-logs/
├── README.md                      # 本文件
├── 001-项目愿景与定位.md          # 项目基本信息
├── 002-市场调研报告.md            # 竞品分析和市场定位
├── 003-技术选型调研.md            # 技术方案调研
├── 004-产品形态分析.md            # 产品形态对比
├── 005-实施方案与阶段规划.md      # 执行计划（由主 agent 生成）
├── 006-架构设计.md                # 技术架构设计
├── conversation-YYYY-MM-DD.md     # 重要对话记录
└── decisions/                     # 重要决策记录
    └── decision-001-xxx.md
```

## 更新机制

- 每次重要讨论后，由专门的 subagent 负责整理和更新文档
- 决策点会记录在 `decisions/` 目录
- 阶段性总结会生成新的编号文档

## 命名规范

- 编号文档：`001-xxx.md` 格式，按创建顺序编号
- 对话记录：`conversation-YYYY-MM-DD.md` 格式
- 决策记录：`decision-001-xxx.md` 格式
