## 一、文档提取技术

### 1.1 PDF 转 Markdown

| 库名 | 语言 | 优点 | 缺点 | 推荐场景 |
|------|------|------|------|----------|
| **pdfplumber** | Python | 精确控制布局，表格提取好 | 速度一般 | 复杂 PDF |
| **Marker** | Python | 结构保持好，支持公式 | 需要计算资源 | 学术 PDF |
| **PyMuPDF** | Python | 性能优异 | 商业用途需授权 | 高性能需求 |
| **pdfminer.six** | Python | 轻量级，纯 Python | 复杂格式支持有限 | 简单 PDF |
| **pdf-parse** | Node.js | 简单 | 功能有限 | 基础需求 |

**推荐**: `Marker` 或 `pdfplumber`（学术/复杂文档），`PyMuPDF`（性能优先）

### 1.2 DOCX 转 Markdown

| 库名 | 语言 | 优点 | 缺点 |
|------|------|------|------|
| **mammoth** | Python/Node.js | 跨平台，API 简单 | 复杂格式可能丢失 |
| **python-docx2markdown** | Python | 基于 python-docx | 高级格式支持有限 |
| **pandoc** | 命令行 | 格式支持最全 | 需要外部依赖 |

**推荐**: `mammoth`（通用），`pandoc`（格式复杂时）

### 1.3 Excel/CSV 转 Markdown

| 库名 | 语言 | 优点 | 缺点 |
|------|------|------|------|
| **pandas** | Python | 功能强大，灵活 | 需要学习曲线 |
| **node-xlsx** | Node.js | 简单 | 功能有限 |

**推荐**: `pandas`（Python 技术栈自然选择）

### 1.4 网页内容提取

| 库名 | 语言 | 优点 | 缺点 |
|------|------|------|------|
| **Mercury Parser** | API | 准确性高 | 有限制 |
| **Readability.js** | Node.js | 开源，Mozilla 背书 | 复杂布局可能不佳 |
| **readability-lxml** | Python | Python 版本 | 更新较慢 |
| **BeautifulSoup** | Python | 灵活，可定制 | 需要自己写提取逻辑 |
| **newspaper3k** | Python | 专为新闻设计 | 非新闻网站效果一般 |

**推荐**: `Mercury Parser API`（准确率优先），`BeautifulSoup`（自定义控制）

## 二、层级索引生成

### 2.1 自动目录生成

| 工具 | 语言 | 描述 |
|------|------|------|
| **markdown-toc** | Node.js | 自动生成 Markdown 目录 |
| **doctoc** | Node.js | 自动生成和更新目录 |
| **markdown-it-toc** | Node.js | markdown-it 插件 |

**推荐**: 自定义实现（更灵活，可结合 AI）

### 2.2 文档结构化

| 工具 | 语言 | 描述 |
|------|------|------|
| **mdast** | Node.js | Markdown 抽象语法树 |
| **remark** | Node.js | Markdown 处理生态系统 |
| **Python-markdown** | Python | Python 的 Markdown 处理器 |

**推荐**: 自定义实现（结合内容分析）

### 2.3 知识图谱/分类

| 工具 | 语言 | 用途 |
|------|------|------|
| **spaCy** | Python | 实体抽取、NLP |
| **NLTK** | Python | 自然语言处理 |
| **Gensim** | Python | 主题建模、文档相似性 |
| **NetworkX** | Python | 图分析、知识图谱 |

**推荐**: `spaCy` + `NetworkX`（实体抽取 + 图谱构建）

| 维度 | Python | Node.js | Go |
|------|--------|---------|-----|
| **文档处理生态** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| **NLP/AI 支持** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| **Web 服务** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **性能** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

| **部署** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

### 最终选择：Python 为主

**理由**:
1. 文档处理领域有更成熟的开源生态
2. 拥有丰富的 OCR、NLP 和 PDF 处理库
3. 便于后续添加 AI 功能进行智能分析
4. 众多现成的开源项目可供参考

## 四、推荐技术栈

### 核心依赖

```python
# PDF 处理
marker-pdf        # PDF 转 Markdown
pdfplumber        # PDF 表格提取

# DOCX 处理
mammoth           # DOCX 转 HTML/Markdown

# 网页提取
mercury-parser    # 或 readability-lxml
beautifulsoup4    # HTML 解析

# NLP 和分类
spacy             # 实体抽取
scikit-learn      # 文本分类

# 知识图谱
networkx          # 图分析

# CLI
typer             # CLI 框架（基于 click）

# 配置管理
pydantic          # 配置验证
```

### 可选依赖

```python
# OCR（扫描件 PDF）
pytesseract       # Tesseract OCR 封装

# 高级 AI 功能
sentence-transformers  # 文本嵌入
chromadb              # 向量数据库（本地）
```
